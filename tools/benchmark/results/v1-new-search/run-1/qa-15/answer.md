**結論**: NablarchのJSPカスタムタグを使用している限り、HTML出力時のエスケープ（XSSの根本的解決の一つ）は自動的に行われます。ただし、カスタムタグが対応するのはHTML属性のエスケープのみであり、JavaScript記述部分や「URLのスキーム制限」「HTMLテキスト入力の許可」などには別途プロジェクト側での対応が必要です。

---

**根拠**:

**カスタムタグが対応すること（IPA 5-(i)）**

カスタムタグはデフォルトで全てのHTML属性についてHTMLエスケープを行います。

変換内容：
```
& → &amp;
< → &lt;
> → &gt;
" → &#034;
' → &#039;
```

これにより「ウェブページに出力する全ての要素に対してエスケープ処理を施す」という根本的解決が可能です。EL式はHTMLエスケープを行わないため、値を出力する場合は `writeタグ` などのカスタムタグを使用してください。

> **重要**: JavaScriptに対するエスケープ処理は提供していません。`<script>` タグのボディや `onclick` 属性など、JavaScriptを記述する部分には動的な値を埋め込まないこと。埋め込む場合はプロジェクトの責任でエスケープ処理を実施してください。

**Nablarchが対応していない項目（プロジェクト対応が必要）**

| IPA対策 | 内容 | Nablarch対応 |
|---------|------|-------------|
| 5-(ii) | URLをhttp://またはhttps://で始まるもののみ許可 | ×（プロジェクト対応） |
| 5-(iii) | `<script>` 要素の内容を動的に生成しない | ×（プロジェクト対応） |
| 5-(iv) | スタイルシートを任意サイトから取り込めるようにしない | ×（プロジェクト対応） |
| 5-(vi) | HTMLテキスト入力許可時の構文解析木による除去 | ×（OSSのHTMLパーサー等を使用） |
| 5-(vii) | 入力HTMLテキストからスクリプト文字列を排除 | ×（プロジェクト対応） |

**保険的対策**

- **入力値チェック（IPA 5-(v)）**: Nablarchの入力値チェック機能（Bean ValidationまたはNablarch Validation）で対応可能です。
- **セキュアハンドラ**: `X-XSS-Protection` ヘッダ等のセキュリティレスポンスヘッダを自動付与します。
- **CSP（Content Security Policy）対応**: `ContentSecurityPolicyHeader` とnonceを組み合わせることでCSPに対応できます。
- **JSP静的解析ツール**: 使用可能なタグ・構文を限定し、カスタムタグ以外のタグ使用によるエスケープ漏れを検出できます。

**注意点**: HTMLテキスト入力を許可する場合（リッチテキストエディタ等）は、Nablarchでは対応していないため、OSSのHTMLパーサーを使用したバリデーション、またはMarkdown+クライアントサイド変換などの方法をプロジェクトで検討してください。
