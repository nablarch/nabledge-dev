**結論**: NablarchはWebSocketを直接サポートする機能を提供していません。

Nablarchの知識ファイルにWebSocketの実装方法に関する情報は含まれていません。

Nablarchはリクエスト/レスポンス型のHTTPベースアーキテクチャ（ウェブアプリケーション、RESTfulウェブサービス）を前提に設計されており、WebSocket通信はNablarchのハンドラキューやフレームワーク管理下では扱われません。

バッチ進捗をブラウザにリアルタイム通知したい場合、Nablarchが公式に案内している代替アプローチは以下です：

- **テーブルをキューとして使ったメッセージング**：バッチがDBに進捗データを書き込み、ウェブ側がポーリングで取得する方式
- 進捗通知のリアルタイム性が必要な場合は、WebSocketサポートはアプリケーションサーバ（Tomcat、WildFlyなど）が提供するJakarta WebSocket（`jakarta.websocket`）の仕様を直接使用することになりますが、その実装はNablarchフレームワーク外の作業になります

この情報は知識ファイルに含まれていません。
