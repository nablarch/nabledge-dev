**結論**: Nablarchのカスタムタグを使うと、HTML出力は自動的にエスケープされるため、IPA基準の「5-(i) 全出力要素へのエスケープ処理」という根本的解決が可能です。ただし、JavaScript部分へのエスケープ、URL出力の検証、`<script>`要素の動的生成禁止などはNablarchが対応しておらず、プロジェクト側での対応が必要です。

---

**根拠**:

**カスタムタグによる自動HTMLエスケープ（5-(i) 根本的解決）**

カスタムタグはデフォルトでHTMLエスケープを行います：

| 文字 | 変換後 |
|------|--------|
| `&` | `&amp;` |
| `<` | `&lt;` |
| `>` | `&gt;` |
| `"` | `&#034;` |
| `'` | `&#039;` |

ただし、**EL式（`${...}`）はHTMLエスケープを行いません**。値を出力する場合は必ず`<n:write>`等のカスタムタグを使用してください。

**Nablarchが対応する範囲（〇）**

| IPA対策項目 | 対応するNablarch機能 | 状況 |
|---|---|---|
| 5-(i) 全出力要素へのエスケープ処理 | カスタムタグ | 〇 |
| 5-(viii) Content-Type charsetの設定 | HTTP文字エンコード制御ハンドラ＋セキュアハンドラ | 〇 |
| 5-(x) XSS対策ブラウザ機能を有効にするレスポンスヘッダ | セキュアハンドラ | 〇（保険的対策） |

**Nablarchが対応しない範囲（×）→プロジェクト対応が必要**

| IPA対策項目 | 状況 |
|---|---|
| 5-(ii) URL出力時に`http://`または`https://`のみ許可 | × |
| 5-(iii) `<script>...</script>`の内容を動的に生成しない | × |
| 5-(iv) スタイルシートへの任意サイトからの取り込み禁止 | × |
| 5-(vi) HTMLテキスト入力を許可する場合のスクリプト排除 | × |

**CSPによる積極的対策（セキュアハンドラ）**

CSP（Content Security Policy）を有効にすることで、XSSインジェクション攻撃の検知・軽減が可能です。セキュアハンドラに`ContentSecurityPolicyHeader`を設定し、カスタムタグのCSP対応機能と組み合わせて使用します。カスタムタグのscript要素にnonceが自動付与されます。

---

**注意点**:

- **JavaScriptへのエスケープはカスタムタグが提供しません**。`onclick`属性やscriptタグ内部に動的な値を埋め込む場合は、プロジェクトの責任でエスケープ処理を実装してください。

- **`rawWrite`タグ・`prettyPrint`タグはHTMLエスケープを行いません**。これらで出力する値がユーザ入力に由来する場合、XSS脆弱性の直接要因となります。`prettyPrint`タグは非推奨です。

- **JSP静的解析ツール**を使用することで、カスタムタグ以外のタグ使用やEL式による直接出力（エスケープ漏れ）を静的に検出できます。
