**結論**: Nablarchのカスタムタグ（Jakarta Server Pagesカスタムタグ）を使用すれば、HTMLへの値出力時にデフォルトでサニタイジング（HTMLエスケープ）が行われ、IPA定義のXSS対策「5-(i) ウェブページに出力する全ての要素に対してエスケープ処理を施す」の根本的解決が可能です。ただし、カスタムタグが対応できないXSS対策項目もあり、それらはプロジェクト側での対応が必要です。

**根拠**:

**カスタムタグが対応するXSS対策**

カスタムタグはデフォルトで以下のHTMLエスケープを全属性に適用します。

| 入力文字 | 変換後 |
|---------|-------|
| `&` | `&amp;` |
| `<` | `&lt;` |
| `>` | `&gt;` |
| `"` | `&#034;` |
| `'` | `&#039;` |

値を出力する際は、EL式（`${...}`）ではなく `<n:write>` などのカスタムタグを使用してください。EL式はHTMLエスケープを行いません。

```jsp
<%-- NG: EL式は HTMLエスケープされない --%>
${user.name}

<%-- OK: カスタムタグはHTMLエスケープする --%>
<n:write name="user.name" />
```

**Nablarchが対応できないXSS対策項目（プロジェクト対応が必要）**

| 対策項目 | 対応状況 | 内容 |
|---------|---------|------|
| 5-(ii) URLの検証 | × | `http://`/`https://`始まりのURLのみ許可する処理はNablarch未提供 |
| 5-(iii) script要素の動的生成禁止 | × | プロジェクトで実装を禁止するルールを設ける |
| 5-(iv) 外部スタイルシートの制限 | × | プロジェクト側で設計・実装が必要 |

**セキュアハンドラによる保険的対策**

Nablarchのセキュアハンドラは、XSS対策を強化する以下のHTTPレスポンスヘッダを自動設定します（IPA 5-(x) 対応）。

- Content-TypeへのMIME Type・文字コード設定（HTTP文字エンコード制御ハンドラ）
- ブラウザのXSS保護機能を有効にするヘッダの出力

**注意点**:

- カスタムタグは**JavaScriptのエスケープを行いません**。`<script>` タグのボディ、`onclick` 属性など、JavaScript内に動的な値（入力データ等）を埋め込む場合は、プロジェクトの責任でJavaScriptエスケープを実施してください。
- `<n:rawWrite>` タグはエスケープなしで出力するため、不特定ユーザが設定可能な値に使用するとXSS脆弱性の直接要因となります。使用には十分な考慮が必要です。
- JSP静的解析ツールを活用することで、カスタムタグ以外のタグ使用によるエスケープ漏れを防止できます。
