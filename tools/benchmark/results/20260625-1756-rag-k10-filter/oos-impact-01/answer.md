### Answer

NablarchにはOAuth2/OpenID Connect（OIDC）の認証機能は標準では組み込まれていません。ただし、**ビジネスサンプル（biz-samples）としてOIDC認証の実装例が提供されています**。

---

#### Nablarch標準機能での認証・認可サポート状況

Nablarchが標準で提供しているのは以下の通りです：

- **認証機能**: 標準提供なし。Nablarch Exampleの実装例（ユーザIDとパスワードによる認証サンプル）を参考にプロジェクトで実装する必要があります。
- **認可チェック**: 標準機能として提供（`認可チェックハンドラ`）。ただし細かいデータ設計が必要なため、システム要件に合う場合に使用してください。

---

#### OIDCサンプルの概要

`guide/biz-samples/biz-samples-12.json` に、**OIDCのIDトークンを用いた認証サンプル**が提供されています。

**想定構成：**
- NablarchのRESTfulウェブサービスをバックエンドとして稼働
- IDトークンはフロントエンド（SPA、モバイルアプリ等）で取得し、バックエンドに送信
- 認証成功時にログインセッションを確立し、その後はステートフルなセッション管理を行う
- 認証処理は業務アクションで実装

**対応サービス：**
- AWS Amazon Cognito ユーザープール
- Microsoft Azure Active Directory B2C（ADB2C）

**ソースコード：** `https://github.com/nablarch/nablarch-biz-sample-all/tree/main/nablarch-oidc`

提供パッケージは `please.change.me.common.oidc.verification` 配下で、Cognitoとadb2cでサブパッケージが分かれています。

> **注意：** このサンプルは2024年9月時点の仕様に基づくため、利用するサービスの最新仕様を確認し、適宜カスタマイズが必要です。

---

#### 関連するセキュリティ対策（ウェブアプリケーション）

OIDC認証導入に合わせて、以下のNablarch標準機能も活用してください：

| セキュリティ対策 | Nablarch機能 | 対応状況 |
|---|---|---|
| セッション管理（セッションID推測困難化・URLパラメータ格納禁止等） | セッションストア | ○ |
| ログイン成功後の新規セッション開始 | Nablarch Example参照 | △（プロジェクト実装必要） |
| クリックジャッキング対策（X-Frame-Options） | セキュアハンドラ | ○ |
| SQLインジェクション対策 | ユニバーサルDAO / JDBCラッパー | ○ |

ウェブアプリケーションのハンドラキューには、`認可チェックハンドラ`・`セッション変数保存ハンドラ`・`セキュアハンドラ`・`CSRFトークン検証ハンドラ`などを組み合わせて構成することが推奨されます。
