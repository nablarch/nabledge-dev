### Answer

ウェブアプリケーションにおけるエラー処理は、主に **HttpErrorHandler** と **HttpResponseHandler** の2つのハンドラが連携して動作します。

---

## エラーハンドリングの全体像

### 1. HttpErrorHandler — 例外の種類に応じたログ出力とレスポンス決定

後続ハンドラ（アクションクラスなど）で発生した例外を捕捉し、例外の種類に応じて以下を決定します。

| 例外クラス | ログレベル | HTTPステータス | 備考 |
|---|---|---|---|
| `NoMoreHandlerException` | INFO | 404 | 処理すべきActionが存在しない → 証跡ログとして記録 |
| `HttpErrorResponse` | なし | レスポンス内容による | バリデーション等の業務例外によるエラーレスポンス → ログ不要 |
| `HttpErrorResponse`（原因が`ApplicationException`） | なし | — | `ApplicationException`のメッセージを`ErrorMessages`に変換してリクエストスコープ（デフォルトキー: `errors`）に設定し、Viewでエラーメッセージを表示できるようにする |
| `Result.Error` | 設定による | `getStatusCode()`の値 | `writeFailureLogPattern`の正規表現がステータスコードにマッチした場合のみFATALログを出力 |
| `StackOverflowError` | FATAL | 500 | データや実装バグに起因する可能性があるため障害扱い |
| 上記以外の例外・エラー | FATAL | 500 | 予期しない例外として障害扱い |

`Result.Error`のログ出力制御のポイント：`writeFailureLogPattern`プロパティに正規表現を設定し、`Error#getStatusCode()`がマッチした場合のみFATALログを出力します。これにより、例えば4xx系は障害ログに出さず5xx系のみFATALにする、といった制御が可能です。

---

### 2. HttpResponseHandler — エラー画面の表示

HttpErrorHandlerが生成したエラーレスポンス（404ページ、エラー画面テンプレートなど）を実際にクライアントへ返す役割を担います。

ただし、**このハンドラ自身の処理中**（サーブレットフォワード時など）に以下が発生した場合は、正常な応答が返せないと判断し、ステータスコード500の固定HTMLを返します：

- サーブレットフォワード時の`ServletException`
- `RuntimeException`およびそのサブクラス
- `Error`およびそのサブクラス

この固定HTMLは設定で変更できません。どんな状況でもこのレスポンスを出してはいけないシステムでは、このハンドラを参考に独自ハンドラを実装する必要があります。

---

### 3. フレームワークのログレベル方針

ログレベルの使い分けは以下の通りです：

| レベル | 出力方針 |
|---|---|
| FATAL/ERROR | 障害監視対象。原則として1件の障害につき1件の障害ログを出力（単一ハンドラから出力） |
| WARN | 障害発生時に連鎖して例外が発生した場合など、障害ログとして出せない例外（例：業務処理とトランザクション終了処理の両方で例外発生時、後者をWARNで出力） |
| INFO | URLパラメータ改竄エラーや認可チェックエラーなど、アプリケーションの実行状況に関連するエラー |
| DEBUG | アプリケーション開発時のデバッグ情報（SQLログ、性能ログ等） |
| TRACE | フレームワーク開発時のデバッグ情報（アプリ開発では使用しない） |

本番運用時はINFOレベルでの出力を想定しており、ログファイルの肥大化を防ぐためにプロジェクトごとに出力内容を規定することが推奨されています。

---

### まとめ：エラー処理のフロー

```
アクション/後続ハンドラ
    ↓ 例外発生
HttpErrorHandler
    ├─ 例外の種類を判定
    ├─ ログ出力（FATAL/INFO/なし）
    ├─ ApplicationExceptionのメッセージをリクエストスコープに設定
    └─ エラーレスポンス（404/500等）を生成
          ↓
HttpResponseHandler
    └─ エラー画面（テンプレート）をクライアントに返却
```
