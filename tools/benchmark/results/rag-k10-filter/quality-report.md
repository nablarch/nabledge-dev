## v3制約に関する見解

RAG実装は現行の v3 スキルアーキテクチャ上で動作しており、検索パラメータ（k値、embedding モデル、segment ランキング）はスキル設計により固定されている。

- **qa-20:s4 未到達**: ベクトル空間でs4よりs5と他セクションが上位にランクされる現象。k値を増やすか、s4 の content を enrichする対処が考えられるが、いずれも v3 制約外（#383 スコープ外）。
- **oos-impact-01 の relevancy flakiness**: OOS シナリオの特性上、ナレッジが直接回答を含まないため LLM が関連知識を広く補完する傾向がある。品質上の問題ではない。
- **impact-08 の faithfulness 低下**: ベースライン比 -0.138。絶対値は 0.816 で閾値 0.7 を上回るが、採点器が context 外の正しい補完を減点する傾向による。

これらはスコアの低下であり、利用者への実害は確認されていない。
