---

**結論**: DB to DBバッチは「都度起動バッチ（DB接続有り）」構成で実装する。`createReader`で集計用SELECTを発行して入力とし、`handle`メソッドで1レコードずつ別テーブルへ`UniversalDao.insert`するのが基本構造。

---

**根拠**:

**ハンドラ構成（最小構成）**

DB接続有りの都度起動バッチに必要なハンドラをコンポーネント定義に設定する：

| No. | ハンドラ | 役割 |
|-----|---------|------|
| 1–2 | ステータスコード変換、グローバルエラー | プロセス終了制御 |
| 3–4 | DB接続管理ハンドラ（初期処理用）、トランザクション制御ハンドラ | 起動時DB接続・コミット |
| 5 | リクエストディスパッチハンドラ | -requestPath でアクション決定 |
| 6 | マルチスレッド実行制御ハンドラ | サブスレッド生成 |
| 7 | DB接続管理ハンドラ（業務処理用） | 業務用DB接続 |
| 8 | トランザクションループ制御ハンドラ | コミット間隔制御 |
| 9 | データリードハンドラ | 1件ずつ読み込んでアクションへ渡す |

コミット間隔はトランザクションループ制御ハンドラの`commitInterval`プロパティで設定する：

```xml
<component class="nablarch.fw.handler.LoopHandler">
  <property name="commitInterval" value="1000" />
</component>
```

**アクションクラスの実装**

`BatchAction`を継承し、`createReader`でSELECT（必要ならJOIN）、`handle`で1件ずつ書き込む：

```java
public class SalesAggregationAction extends BatchAction<SalesDto> {

    @Override
    public Result handle(SalesDto inputData, ExecutionContext ctx) {
        // 集計結果エンティティを組み立てて別テーブルに登録
        SalesSummary summary = BeanUtil.createAndCopy(SalesSummary.class, inputData);
        UniversalDao.insert(summary);
        return new Result.Success();
    }

    @Override
    public DataReader<SalesDto> createReader(ExecutionContext ctx) {
        // DBからの読み込みはここで行う（handleでSELECT追加発行しない）
        return new DatabaseRecordReader()
                .setStatement(/* SQLIDを指定 */)
                .setBean(SalesDto.class);
    }
}
```

集計が複雑な場合は`UniversalDao.findAllBySqlFile`でSQLファイルを使う：

```java
UniversalDao.findAllBySqlFile(SalesDto.class, "AGGREGATE_SALES");
```

大量データを扱う場合は遅延ロードでメモリを節約する：

```java
try (DeferredEntityList<SalesDto> records =
        (DeferredEntityList<SalesDto>) UniversalDao.defer()
                .findAllBySqlFile(SalesDto.class, "AGGREGATE_SALES")) {
    // handleに渡す前の前処理など
}
```

**DB to DBパターンの特性**

- 入力はSELECT結果セットの各レコード
- 1レコードの処理中の更新は全て同一トランザクション下で実行されるため、障害発生時に不整合が発生しない

---

**注意点**:

- **N+1問題**: `handle`メソッド内で追加SELECTを発行しない。必要なデータは`createReader`でJOINして1回のSQLで取得する。10,000件の場合、N+1があると10,001件のSQLが発行される
- **自前ループ禁止**: `handle`メソッド内で`for`ループを使ってSELECT+UPDATE/INSERTをしない。フレームワーク管理外のループはコミットが実行されずトランザクションログを逼迫する。その処理はSQLを`createReader`に移して解決する
- **遅延ロードとトランザクション**: `DeferredEntityList`使用中にトランザクション制御が行われるとカーソルがクローズされる場合がある（RDBMSによる）。コミット間隔とカーソル挙動の整合を確認すること
