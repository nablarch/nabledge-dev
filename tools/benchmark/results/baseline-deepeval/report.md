## Q4. このベースラインは退行検出に使えるか？

**条件付きでYES**。ただし以下の留意点がある。

**使える根拠**:
- 30シナリオ中7件（impact-08 / oos-impact-01 / qa-07 / qa-09 / qa-10 / qa-14 / qa-15）が3指標全て3 run通過（安定シナリオ）。これらは退行検出の信頼性が高い。
- `baseline.json` に mean±stddev が記録されており、`mean - 2σ` ルールで偽陽性を抑制した退行検出が可能。
- フラキーシナリオ（23件）は退行扱いせず別報告するため、評価器の揺らぎで誤検知しない。

**留意点**:
- qa-05 / qa-12a は評価基準（must.facts）の記述の問題でスコアが不安定。退行が出てもスキルの劣化とは断定できない。これら2件の must.facts を修正してから再取得するとベースラインの精度が上がる。
- impact-03 / pre-01 は評価器が質問スコープの解釈でブレており、3 run全て閾値未達。この2件についての退行検出は現状では意味をなさない。
