### Answer

**結論**: NablarchにはWebSocketを直接サポートする機能・ハンドラは提供されていません。

Nablarchのウェブアプリケーションは Jakarta Servlet API（サーブレットフィルタ＋ハンドラキュー）を前提とした設計であり、WebSocket接続の確立・メッセージ送受信に関するフレームワーク機能は知識ファイルの対象範囲外です。

**根拠**: Nablarch の基本方針として「OSSは使用しない」があり、独自実装を中心に構成されています。知識ファイルに含まれる Nablarch のウェブアプリケーション機能一覧（機能詳細）には、WebSocket に相当する機能は列挙されていません。移行ガイドの仕様対応表に "Jakarta WebSocket (javax.websocket → jakarta.websocket)" という名称が登場しますが、これは Java EE/Jakarta EE の仕様名の対応を示したものに過ぎず、Nablarch が WebSocket をサポートすることを意味するものではありません。

**注意点**:
- バッチの進捗をリアルタイムにブラウザへ通知する手段として、Nablarch が提供する代替アプローチは知識ファイルに記載がありません。
- アプリケーションサーバー（Tomcat、WildFly 等）が Jakarta WebSocket 仕様を実装しているため、Jakarta EE 標準の `@ServerEndpoint` を使った WebSocket エンドポイントをアプリケーション内に自前実装することは技術的に可能ですが、それは Nablarch のフレームワーク外の実装になります。その具体的な方法は知識ファイルの対象範囲外です。
