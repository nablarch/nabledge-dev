### Answer

**結論**: DB to DBパターンの都度起動バッチとして実装します。`BatchAction<T>`を継承した業務アクションクラスを作成し、`createReader()`で`DatabaseRecordReader`（集計・入力SELECT SQL）を返し、`handle()`で1レコードずつ`UniversalDao.insert()`で結果テーブルに書き込みます。

---

**根拠**:

**起動方式とパターンの選択**

- 日次・月次などの定期処理 → **都度起動バッチ**
- DBから読み込んで別テーブルへ書く → **DB to DBパターン**（都度起動 × DB to DB = ○）
- 1レコードの処理中の更新は全て同一トランザクション下で実行されるため、障害発生時でも不整合が発生しません

**ハンドラキュー（DB接続有りの業務処理部分）**

| No. | ハンドラ | 役割 |
|-----|---------|------|
| 7 | データベース接続管理ハンドラ（業務処理用） | DB接続の取得・解放 |
| 8 | トランザクションループ制御ハンドラ（LoopHandler） | コミット間隔制御・ループ継続 |
| 9 | データリードハンドラ | 1件ずつ読み込みアクションへ渡す |

**業務アクションの実装パターン**

```java
public class AggregationBatchAction extends BatchAction<集計対象エンティティ> {

    @Override
    public Result handle(集計対象エンティティ inputData, ExecutionContext ctx) {
        // inputDataを元に集計結果エンティティを組み立ててINSERT
        集計結果エンティティ result = new 集計結果エンティティ();
        // ... inputDataから値を設定
        UniversalDao.insert(result);
        return new Result.Success();
    }

    @Override
    public DataReader<集計対象エンティティ> createReader(ExecutionContext ctx) {
        // 入力データを取得するSQLをここで発行する（DatabaseRecordReaderを使用）
        // 集計が必要なデータはここでJOINやGROUP BYを使って1回のSQLで取得する
    }
}
```

**コミット間隔の設定（LoopHandler）**

```xml
<component class="nablarch.fw.handler.LoopHandler">
  <!-- コミット間隔に1000を指定 -->
  <property name="commitInterval" value="1000" />
</component>
```

---

**注意点**:

- **N+1問題を避ける**: `handle()`内で追加のSELECTを発行してはいけません。処理対象が10,000件であれば10,001件のSQLが発行され、深刻な性能劣化を招きます。集計に必要なデータは`createReader()`のSQLでJOINして一括取得してください。
- **自前ループ禁止**: `handle()`内で自前ループを書かないでください。フレームワークのループは`commitInterval`間隔でコミットしますが、自前ループではコミットされず、更新件数が増えるとトランザクションログを逼迫します。`createReader()`でSQLを発行し、フレームワークのループに委ねてください。
