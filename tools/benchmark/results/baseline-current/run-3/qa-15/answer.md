**結論**: Nablarchのカスタムタグを使えばHTML出力時のサニタイジング（HTMLエスケープ）は自動的に行われ、IPA定義のXSS根本的解決策の一つ（5-(i) ウェブページに出力する全ての要素にエスケープ処理を施す）に対応できます。ただし、カバーできない対策項目が複数あり、それらはプロジェクト側での実装が必要です。

**根拠**:

**1. カスタムタグのサニタイジング（5-(i)：根本的解決 〇）**

Nablarchのカスタムタグ（`<n:write>` 等）はデフォルトでHTMLエスケープを行います。変換内容は以下の通りです：
- `&` → `&amp;`
- `<` → `&lt;`
- `>` → `&gt;`
- `"` → `&#034;`
- `'` → `&#039;`

セキュリティチェックリストには「Nablarchのカスタムタグはサニタイジングを行います。これによりNablarchのカスタムタグを使った場合には5-(i)の根本的解決が可能です」と明記されています。

ただし、JSPのEL式（`${...}`）はHTMLエスケープを行いません。値の出力には必ずカスタムタグを使用してください。

**2. エスケープを行わないタグは注意が必要**

`<n:rawWrite>` タグはHTMLエスケープなしで出力するため、不特定ユーザが任意に設定できる値に使用するとXSS脆弱性の直接の原因になります。`<n:prettyPrint>` は現在非推奨です。

**3. JavaScriptコンテキストへの動的埋め込みは非対応（5-(iii)：×）**

カスタムタグはJavaScriptエスケープを提供していません。`<script>` タグのボディや `onclick` 属性など、JavaScriptを記述する部分には動的な値を埋め込まないことが原則です。埋め込む場合はプロジェクト側でエスケープ処理を実装してください。

**4. Nablarchが対応しているXSS対策項目のまとめ**

| 対策（IPA 5-(*) ） | 対応状況 | 説明 |
|---|---|---|
| 5-(i) HTMLエスケープ | 〇（カスタムタグ） | カスタムタグ使用で根本解決 |
| 5-(ii) URL出力の制限 | × | プロジェクト対応が必要 |
| 5-(iii) script要素の動的生成禁止 | × | プロジェクト対応が必要 |
| 5-(iv) スタイルシート取り込み制限 | × | プロジェクト対応が必要 |
| 5-(v) 入力値チェック | 〇（入力値チェック機能） | 保険的対策として対応 |
| 5-(vi) HTMLテキスト入力時のパース | × | OSS HTMLパーサ等での対応を検討 |
| 5-(vii) スクリプト文字列の排除 | × | プロジェクト対応が必要 |
| 5-(viii) Content-Type charset設定 | 〇（HTTP文字エンコード制御ハンドラ） | 全体対策として対応 |
| 5-(ix) HttpOnly + TRACEメソッド無効化 | × | プロジェクト対応が必要 |
| 5-(x) XSSブラウザ機能有効化ヘッダ | 〇（セキュアハンドラ） | X-XSS-Protection等を自動設定 |

**5. セキュアハンドラによる保険的対策（5-(x)：〇）**

`SecureHandler` はデフォルトで `X-XSS-Protection: 1; mode=block` などのセキュリティヘッダを設定し、ブラウザ側のXSS防御機能を有効化します。CSP（Content Security Policy）にも対応しており、`ContentSecurityPolicyHeader` を組み合わせることでXSS攻撃の影響を軽減できます。

**6. JSP静的解析ツールによるエスケープ漏れ検出**

Nablarchが提供するJSP静的解析ツール（`Jakarta Server Pages静的解析ツール`）を使用すると、JSPで使用を許可するタグ・構文を規定し、カスタムタグ以外の使用（EL式の直接出力等）によるサニタイジング漏れをビルド時に検出できます。

**注意点**:
- EL式（`${...}`）でそのまま値を出力するとエスケープされないため、必ず `<n:write>` 等のカスタムタグを使用すること
- `<n:rawWrite>` の使用は慎重に（XSS脆弱性の直接原因になりえる）
- JavaScriptへの動的値埋め込みはNablarchではカバーされないため、設計段階から回避すること
- CSP対応をする場合、onclick属性へのインラインJavaScript記述は `unsafe-inline` となりCSP効果が下がるため、外部スクリプトまたはnonce指定のscript要素を使用すること
