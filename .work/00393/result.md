# ベンチマーク精度検証レポート

## 結論

**現行のベンチマーク設定は問題のある回答をほぼ見逃さないが、問題のない回答を誤って NG と判定する頻度が高い。**

| 指標 | 誤検知率 | 見逃し率 |
|------|----------|----------|
| answer_correctness | 44.4%（4/9 件） | 0.0%（0/93 件） |
| answer_relevancy | 60.9%（14/23 件） | 0.0%（0/79 件） |
| faithfulness | 74.3%（26/35 件） | 1.5%（1/67 件） |

- **誤検知**: DeepEval が NG と判定したが、実際には問題なかった件
- **見逃し**: DeepEval が OK と判定したが、実際には問題があった件

## 背景と検証方法

### 対象システム

**nabledge-6** は、Java フレームワーク Nablarch 6 に関する質問に回答する AI アシスタント（Claude Code スキル）である。ナレッジファイル（JSON 形式のドキュメント）を参照して回答を生成する。

### 現行ベンチマーク設定

**DeepEval** の RAG 評価メトリクスを使って回答品質を自動評価している。各シナリオで以下の 4 つを `LLMTestCase` に渡し、3 指標を測定する。

| `LLMTestCase` のフィールド | 内容 |
|---|---|
| `input` | ユーザーの質問文 |
| `actual_output` | nabledge-6 が生成した回答 |
| `expected_output` | 回答が必ず含むべき事実（1〜数件） |
| `retrieval_context` | nabledge-6 が参照したナレッジセクションの本文 |

使用する DeepEval メトリクスと閾値は以下の通り。スコアが閾値を下回ると NG（失敗）と判定する。本レポートで検証するのは、この指標・閾値・インプット構成の組み合わせによる判定精度である。

| 指標 | DeepEval メトリクス | 測定内容 | 合格閾値 |
|------|---------------------|----------|----------|
| answer_correctness | `GEval` | `actual_output` が `expected_output` の全事実を含んでいるか | スコア ≥ 0.99 |
| answer_relevancy | `AnswerRelevancyMetric` | `actual_output` が `input` に関連しているか | スコア ≥ 0.95 |
| faithfulness | `FaithfulnessMetric` | `actual_output` が `retrieval_context` と矛盾しないか | スコア ≥ 0.99 |

### 検証方法

現行ベンチマークの判定精度を調べるため、別の AI エージェント（以下「**AI 照合**」）が各回答とナレッジを独立に読んで合否を判定した。AI 照合はベンチマークのスコアを参照せず、回答とナレッジのみを根拠とした。この AI 照合の結果をグラウンドトゥルース（正解）として、ベンチマークの判定と照らし合わせた。

**対象データ**: `tools/benchmark/results/20260616-1214-fullbench-classes-v6/` の run-1〜run-3（34 シナリオ × 3 回 = 102 件）  
**照合日**: 2026-07-07

## 指標ごとの詳細

### answer_correctness（回答が期待される事実を含んでいるか）

| | DeepEval=OK | DeepEval=NG |
|---|---|---|
| **AI 照合=OK** | 93 件（両者合意・問題なし） | **4 件（誤検知）** |
| **AI 照合=NG** | 0 件（見逃しなし） | 5 件（両者合意・問題あり） |

**誤検知の原因**: 現行ベンチマークは「回答に補足情報が多い」「参照事実と表現が異なる」といった理由で NG と判定するが、参照すべき事実は実際に回答に含まれている。

誤検知の例:
- [run-1-qa-12.md](checks/run-1-qa-12.md) — 期待される事実は回答に含まれているが、補足情報が多いとして NG
- [run-3-review-09.md](checks/run-3-review-09.md) — 事実は含まれているが表現が異なるとして NG

### answer_relevancy（回答が質問に関連しているか）

| | DeepEval=OK | DeepEval=NG |
|---|---|---|
| **AI 照合=OK** | 79 件（両者合意・問題なし） | **14 件（誤検知）** |
| **AI 照合=NG** | 0 件（見逃しなし） | 9 件（両者合意・問題あり） |

**誤検知の原因**: nabledge-6 の回答末尾には `参照: libraries-xxx.json:s9` という形式の出典メタデータが付与される。現行ベンチマークはこれを「質問と無関係なコンテンツ」と解釈してスコアを下げる。AI 照合では回答の本文が質問に直接答えており問題なしと判断した。誤検知 14 件中 13 件がこのパターンに該当する。

誤検知の例:
- [run-1-impact-01.md](checks/run-1-impact-01.md) — 本文は質問に直接関連しているが、末尾の出典メタデータを減点対象として NG
- [run-1-qa-03.md](checks/run-1-qa-03.md) — 同上

### faithfulness（回答がナレッジと矛盾しないか）

| | DeepEval=OK | DeepEval=NG |
|---|---|---|
| **AI 照合=OK** | 66 件（両者合意・問題なし） | **26 件（誤検知）** |
| **AI 照合=NG** | 1 件（**見逃し**） | 9 件（両者合意・問題あり） |

**誤検知の原因**: 現行ベンチマークは「ナレッジに記載された情報を回答が省略している」ことを矛盾と解釈する。しかし省略は矛盾ではない。AI 照合では「回答に含まれる記述がナレッジと矛盾しているか」のみを基準としたため、判定が分かれた。

誤検知の例:
- [run-1-impact-01.md](checks/run-1-impact-01.md) — コンストラクタの引数パターンを一部省略しているとして NG。ナレッジとの矛盾はない。

見逃しの例（1 件）:
- [run-2-qa-11.md](checks/run-2-qa-11.md) — ベンチマーク=OK だが、回答にナレッジに存在しない記述が含まれている。
