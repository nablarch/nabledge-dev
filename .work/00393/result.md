# ベンチマーク精度検証レポート

## 結論

**現行のベンチマーク設定は問題のある回答をほぼ見逃さないが、問題のない回答を誤って NG と判定する頻度が高い。**

| 指標 | 誤検知率 | 見逃し率 |
|------|----------|----------|
| answer_correctness | 88.9%（8/9 件） | 0.0%（0/93 件） |
| answer_relevancy | 60.9%（14/23 件） | 0.0%（0/79 件） |
| faithfulness | 94.3%（33/35 件） | 1.5%（1/67 件） |

- **誤検知**: ベンチマークが NG と判定したが、AI 照合では問題なかった件
- **見逃し**: ベンチマークが OK と判定したが、AI 照合では問題があった件

## 背景と検証方法

### 対象システム

**nabledge-6** は、Java フレームワーク Nablarch 6 に関する質問に回答する AI アシスタント（Claude Code スキル）である。ナレッジファイルを参照して回答を生成する。

### 現行ベンチマーク設定

**DeepEval** の RAG 評価メトリクスを使って回答品質を自動評価している。各シナリオで以下の 4 種類の情報を評価器に渡し、3 指標を測定する。

| 評価器への入力 | 内容 |
|---|---|
| 質問文 | ユーザーの質問 |
| 回答 | nabledge-6 が生成した回答 |
| 期待事実 | 回答が必ず含むべき事実（数件） |
| 参照コンテキスト | nabledge-6 が参照したナレッジセクションの本文（ページ全文ではなく、選択されたセクションのみ） |

スコアが閾値を下回ると NG（失敗）と判定する。本レポートで検証するのは、この指標・閾値・入力構成の組み合わせによる判定精度である。

| 指標 | DeepEval メトリクス | 測定内容 | 合格閾値 |
|------|---------------------|----------|----------|
| answer_correctness | `GEval` | 回答が期待事実をすべて含んでいるか | スコア ≥ 0.99 |
| answer_relevancy | `AnswerRelevancyMetric` | 回答が質問に関連しているか | スコア ≥ 0.95 |
| faithfulness | `FaithfulnessMetric` | 回答が参照コンテキストと矛盾しないか | スコア ≥ 0.99 |

#### 入出力の例（代表 5 件）

| 質問 | 期待事実 |
|---|---|
| 大量データを検索するとメモリが足りなくなる。1件ずつ読み込む方法はないか？ | UniversalDao.deferメソッドで遅延ロードを使用する / DeferredEntityListのcloseメソッドを呼び出す必要がある |
| 業務トランザクションとは別のトランザクションでSQLを実行する方法はあるか？ロールバックされても別トランザクションの更新は残したい | SimpleDbTransactionManagerを使って個別トランザクションを定義する |
| 入力画面と確認画面のJSPを共通化して実装を減らす方法はあるか？ | confirmationPageタグで確認画面のJSPに入力画面のJSPへのパスを指定して共通化する |
| バッチ処理の進捗状況をWebSocketでリアルタイムにブラウザへ通知したい。NablarchでWebSocketを使う方法はあるか？ | NablarchにはWebSocketのサポートがないと回答している |
| REST APIでURLパスの一部を受け取ったり、検索条件をURL末尾のパラメータで渡す実装はどう書く？ルーティングの設定も含めて確認したい | パスパラメータはルーティング設定で名前を定義しリソースクラスで受け取る / クエリーパラメータはJaxRsHttpRequestから取得する |

### 検証方法

検証は 2 ステップで実施した。

**ステップ 1: AI 照合**  
別の AI エージェント（以下「**AI 照合**」）が 102 件を 1 件ずつ独立に読んで合否を判定した。AI 照合はベンチマークのスコアを参照せず、回答とナレッジのみを根拠とした。指標ごとの評価観点と参照した情報は以下の通り。

| 指標 | AI 照合の観点 | AI 照合が参照した情報 |
|------|--------------|-------------------|
| answer_correctness | 期待事実が回答に含まれているか | 回答・期待事実 |
| answer_relevancy | 回答に質問と無関係な内容が含まれているか | 回答・質問文・参照ナレッジページ全文 |
| faithfulness | 回答がナレッジの記述と矛盾しているか | 回答・参照ナレッジページ全文 |

**ステップ 2: 人間による最終判断**（未実施）  
AI 照合で NG とした件について、人間が回答とナレッジを突合してユーザーへの実害有無を確認し最終判定を下す。本レポートはステップ 1 の結果に基づく暫定値であり、ステップ 2 完了後に「要・実害確認」の件数が変わる可能性がある。

**対象データ**: `tools/benchmark/results/20260616-1214-fullbench-classes-v6/` の run-1〜run-3（34 シナリオ × 3 回 = 102 件）  
**照合日**: 2026-07-07

## 指標ごとの詳細

### answer_correctness（回答が期待事実を含んでいるか）

| | ベンチマーク OK | ベンチマーク NG |
|---|---|---|
| **AI 照合 OK** | 93 件（両者合意・問題なし） | **8 件（誤検知）** |
| **AI 照合 NG** | 0 件（見逃しなし） | 1 件（要・実害確認） |

**誤検知の原因**: ベンチマークは「回答に補足情報が多い」「参照事実と表現が異なる」といった理由で NG と判定するが、参照すべき事実は実際に回答に含まれている。

誤検知の例:
- [run-1-qa-12.md](checks/run-1-qa-12.md) — 期待事実は回答に含まれているが、補足情報が多いとして NG
- [run-3-review-09.md](checks/run-3-review-09.md) — 事実は含まれているが表現が異なるとして NG

### answer_relevancy（回答が質問に関連しているか）

| | ベンチマーク OK | ベンチマーク NG |
|---|---|---|
| **AI 照合 OK** | 79 件（両者合意・問題なし） | **14 件（誤検知）** |
| **AI 照合 NG** | 0 件（見逃しなし） | 9 件（要・実害確認） |

**誤検知の原因**: nabledge-6 の回答末尾には `参照: libraries-xxx.json:s9` という形式の出典メタデータが付与される。ベンチマークはこれを「質問と無関係なコンテンツ」と解釈してスコアを下げる。AI 照合では回答の本文が質問に直接答えており問題なしと判断した。誤検知 14 件中 13 件がこのパターンに該当する。

誤検知の例:
- [run-1-impact-01.md](checks/run-1-impact-01.md) — 本文は質問に直接関連しているが、末尾の出典メタデータを減点対象として NG
- [run-1-qa-03.md](checks/run-1-qa-03.md) — 同上

### faithfulness（回答がナレッジと矛盾しないか）

| | ベンチマーク OK | ベンチマーク NG |
|---|---|---|
| **AI 照合 OK** | 66 件（両者合意・問題なし） | **33 件（誤検知）** |
| **AI 照合 NG** | 1 件（**見逃し**） | 2 件（要・実害確認） |

**誤検知の原因**: ベンチマークは「ナレッジに記載された情報を回答が省略している」ことを矛盾と解釈する。しかし省略は矛盾ではない。AI 照合では「回答に含まれる記述がナレッジと矛盾しているか」のみを基準としたため、判定が分かれた。

誤検知の例:
- [run-1-impact-01.md](checks/run-1-impact-01.md) — コンストラクタの引数パターンを一部省略しているとして NG。ナレッジとの矛盾はない。

見逃しの例（1 件）:
- [run-2-qa-11.md](checks/run-2-qa-11.md) — ベンチマーク OK だが、回答にナレッジに存在しない記述が含まれている。
