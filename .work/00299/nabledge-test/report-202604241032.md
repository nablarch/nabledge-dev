## 🔬 仮説と改善提案

### 仮説 1: qa-001 シナリオを 2 件に分割すれば benchmark 安定

- 証拠: trial 1/2 が 5/8、trial 3 のみ 8/8。Sonnet が code タグ専用 vs 汎用 select 併記の 2 スコープで揺らぐ
- 検証: `qa-001a: コード値プルダウン` + `qa-001b: 汎用セレクトタグ` に分割
- 予測: 両方とも 100% ±0% に収束

### 仮説 2: ca-003 Overview の英名欠落は Overview テンプレート制約で解消

- 証拠: 同ドキュメントの Nablarch Framework Usage には `### BusinessDateUtil` が存在。Overview 段落だけ日本語要約で英名が落ちる
- 検証: `nabledge-6/workflows/code-analysis.md` Overview テンプレートに「主要 Nablarch クラスは英名を backtick で必ず併記」制約追加
- 予測: ca-003 36/37 → 37/37

### 仮説 3: `.nabledge/` を trial-scoped にすれば benchmark CA output 競合解消

- 証拠: ca-003 3 trial が同一ファイルに書き込み、最後の trial 以外の output が喪失
- 検証: `nabledge-test-runner` が `WORKSPACE_DIR` 経由で trial-scoped な output dir を target skill に渡す (または nabledge-6 の record-start で RUN_ID-TRIAL を差し込む)
- 予測: 各 trial の output が独立保存され、trial 間の差を正確に計測できるように

### 仮説 4: 600 秒 watchdog stall は Skill-in-subagent の相互作用

- 証拠: qa-002、ca-003 t2 が共に "Skill 呼び出し後 / 次ステップへの遷移" で停止。retry では再現せず
- 検証: 次回 baseline でも stall 発生率を追跡。33% (2/6 benchmark trials) が続くようなら Claude Code 側の issue 起票対象
- 予測: 偶発的なら次回は 0-1 件、定常的なら再度 2 件程度
