# QO2 Bias-Avoidance QA Review (Z-1 r7)

Target: `check_json_docs_md_consistency` (QO2 portions) in
`tools/rbkc/scripts/verify/verify.py` vs spec §3-3 QO2.

Spec clause (§3-3, l.295-297):
> **QO2 本文整合性**:
> - JSON top-level `content` が docs MD の `#` 見出し直下に完全一致で含まれている
> - JSON 各セクションの `content` が docs MD に完全一致で含まれている

Quality gate (§2-1 ゼロトレランス): verbatim means verbatim; no implicit skip,
no tolerance.

---

## Findings

### F1. Asset-link rewrite test is circular (hand-coded expected, not derived from `docs._rewrite_asset_links`)

Spec §3-3 l.289:
> docs MD は JSON の人間可読レンダリングであるため、両者は完全に一致しなければならない。

The symmetry QO2 relies on is:
*verify's rewrite output == docs.py's rewrite output*.

In `test_pass_assets_link_rewrite_symmetric`
(`tests/ut/test_verify.py:228-249`), the docs MD string is hand-coded:

```python
docs = "# T\n\n## 図\n\n![図](../../knowledge/assets/img.png)\n"
```

The rewritten URL `../../knowledge/assets/img.png` is not generated by
calling `scripts.create.docs._rewrite_asset_links(...)`; the test author
computed it and pasted the result. Because
`verify._apply_asset_link_rewrite` is a **separate, duplicate
implementation** of the same logic (it does not import from
`scripts.create.docs`), the test only proves that verify's copy agrees
with the author's mental model — not that verify agrees with the actual
docs writer.

If `docs._rewrite_asset_links` drifts (e.g. adds URL-encoding, trims
trailing `/`, normalises separators, etc.) and `verify._apply_asset_link_rewrite`
stays put (or vice versa), v6 docs MD will be produced with one form
while QO2 expects the other — yet this test will still pass because
both the docs string and the expected rewrite are hand-authored in the
test file.

Proposed fix: either

1. have verify import `_rewrite_asset_links` from `scripts.create.docs`
   so there is one implementation, or
2. in the test, call `docs._rewrite_asset_links(json_content, docs_md_path, kdir)`
   to produce the expected docs body, and assert verify passes against
   that generated body.

Option 1 is stricter (eliminates the duplicate-drift root cause
entirely); option 2 at minimum restores non-circular verification.

### F2. Silent skip of empty-content sections (`if not content: continue`)

`verify.py:184-185`:

```python
if not content:
    continue
```

Spec §3-3 QO2 l.297 requires *"JSON 各セクションの `content` が docs MD
に完全一致で含まれている"* — unconditional. The spec does not sanction
"skip if empty". It is true that `"" in any_text` would trivially pass,
so the observable behaviour on this path happens to match spec — but
the structural pattern here is the same "silent skip without spec
backing" root cause flagged in `.claude/rules/rbkc.md`
(*"JSON section with empty content — skip or match?" → Match. Spec §3-3
requires verbatim containment.*).

That rule explicitly names this exact QO2 case and requires **match,
not skip**. The current code violates that directive regardless of
whether the behaviour currently diverges.

Proposed fix: remove the `if not content: continue` guard and fall
through to the `expected not in docs_md_text` check. An empty content
trivially satisfies the containment check without a special case.

### F3. Top-content `##`-in-fence region-bounding not covered by a PASS test

`verify.py:165-178` computes the "top region" by masking fenced code
blocks and taking `docs_md_text[start:end]` where `end` is the first
**post-mask** `##` offset. This logic is load-bearing: if the top
`content` contains `##` inside a fenced block, the region must not be
truncated at that fake `##`.

Tests cover the H1/H2 scan side (`TestCheckJsonDocsMdConsistency_QO1::
test_pass_fake_h2_inside_backtick_fence` at l.178-184) but
`TestCheckJsonDocsMdConsistency_QO2` has no test where the JSON
top-level `content` itself contains a fenced `##`. A regression that
forgot to mask fences on the QO2 side (e.g. scanning against
`docs_md_text` instead of `masked`) would not be caught by any existing
test — `test_pass_section_content_with_fenced_code` tests section
content, which uses plain `in` on full `docs_md_text` and does not
exercise the region-bounding code path at all.

Proposed fix: add a PASS test where `data["content"]` contains
````markdown
\n```md\n## not a heading\n```\n````
and docs MD places that fenced block between `#` and the first real
`##`. Add a FAIL twin where the fenced content is missing from docs.

### F4. `_apply_asset_link_rewrite` is a full re-implementation of `docs._rewrite_asset_links`, not a mirror

`verify.py:72-97` duplicates `scripts/create/docs.py:43-69` byte-for-byte
in structure (same `_MD_LINK_RE`, same `os.path.relpath`, same
`assets/`-prefix test). verify imports nothing from `scripts.create.docs`.

Per `.claude/rules/rbkc.md`:
> verify must never import from or depend on RBKC implementation modules
> (converters, resolver, run, etc.)

So importing is forbidden — which means the drift risk is structural.
The current docstring acknowledges the mirror relationship
(*"Mirror docs.py's asset-link rewrite so QO2 compares like-for-like"*)
but there is no mechanism — no test, no assertion, no shared
spec-derived oracle — that detects when the two copies diverge.

This turns F1 from a test-design issue into a systemic gap: verify
claims to mirror docs, the tests claim to verify the mirror, but
neither ever cross-checks against the docs-side implementation.

Proposed fix: add a verify unit test that imports
`scripts.create.docs._rewrite_asset_links` **only inside the test**
(not inside verify.py), runs both transforms on a matrix of inputs
(nested dirs, multiple assets, no-assets content, mixed image+link,
trailing-slash, query strings) and asserts equal output. This keeps
verify.py decoupled from RBKC internals while catching mirror drift
at test time.

## Observations (not blocking)

- `_MD_LINK_RE` is independently defined in verify.py and docs.py.
  Same root cause as F4 — covered by F4's proposed test.
- QO2 section containment uses `expected not in docs_md_text` — it does
  not require the content to appear under its own `##` heading. The
  spec wording *"docs MD に完全一致で含まれている"* does not require
  positional containment, so this is per-spec. A malicious/adversarial
  docs MD that placed section B's content under section A's heading
  would pass QO2 but fail QO1 (section-title order), so the pair is
  sound.
- `_H2_RE` matches both `##` and `###`, so nested sub-section headings
  participate in QO1 order comparison — orthogonal to QO2.
- Whitespace-only divergence FAIL (`test_fail_whitespace_only_diff`,
  l.273-281) correctly enforces verbatim (no normalisation).
