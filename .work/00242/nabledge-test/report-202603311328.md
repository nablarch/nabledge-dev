## 🔬 仮説と改善提案

### 仮説1: qa-001/qa-004 の期待値は nabledge-1.4 スコープ外

**根拠**: `n:select` と `@UseToken` は nabledge-1.4 の知識ファイルに存在しないか、主方式として扱われていない。v5/v6 では既に修正済み（`n:codeSelect`/`codeId` に変更）。

**検証方法**: nabledge-1.4 の知識ファイルで `n:select`, `@UseToken` を grep して存在確認。

**期待結果**: 期待値を `@OnDoubleSubmission`/`useToken` に整合させることで qa-004 検出率が 100% になる可能性がある。qa-001 の `n:select` は削除候補。

### 仮説2: CA Overview のクラス名出力は skill プロンプトで誘導可能

**根拠**: ca-002 trial 2 では `ValidatableFileDataReader` が Overview に出現したが（85.3%）、trial 1/3 では出現しなかった（79.4%/82.4%）。出力は確率的であり、プロンプトで明示的に指示すれば安定する可能性がある。

**検証方法**: コード分析テンプレートの Overview セクション指示に「主要な Nablarch クラス名を含めること」を追加して再測定。

**期待結果**: Overview 検出率が安定し、CA 全体の検出率が +5〜10% 改善する可能性がある。

### 仮説3: `getValidatorAction` は Sequence Diagram への追加で検出可能

**根拠**: `getValidatorAction` はフレームワーク統合の重要ポイントだが、現状では Components セクションにのみ記述される。Sequence Diagram に「`getValidatorAction()` → `FileLayoutValidatorAction` 返却」のメッセージを加えることで検出可能になる。

**検証方法**: コード分析テンプレートの Sequence Diagram 生成指示に「ファクトリメソッド/初期化メソッドを含めること」を追加。

**期待結果**: ca-002 の Sequence Diagram messages 検出率が +1/3 改善（2/3 → 3/3）。
