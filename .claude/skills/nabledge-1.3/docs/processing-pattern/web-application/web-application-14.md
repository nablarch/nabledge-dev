## Firefoxで2重サブミット防止が効かないことがある問題

`n:submitLink`で生成されたリンクを素早くダブルクリックした場合、Firefoxの古いバージョン（3.x系を含む）でまれに2回連続リクエストが送信される事象が発生する。Chrome/Safari/IE6では発生しない。

Firefoxバージョン9.01以降にアップデートすることで解消する。

> **重要**: 二重サブミット防止機能はサーバサイドのトークンチェックと併用することが前提となっている。そのため、本事象が発生した場合でも業務処理が2重実行されることはない。
