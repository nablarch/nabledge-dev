# 序 : Nablarch Application Framework (NAF)とは ?

## NAFの設計思想

NAFはTISが開発したJavaベースの基幹システム向けアプリケーションフレームワーク。設計思想は以下の3点。

## 1. エンタープライズシステムに必要な機能要素を包括的に提供

真の「フルスタック」フレームワークとして以下を包含：

- **外部入力**: 画面オンライン入力、バッチ入力、他システムからの要求電文
- **外部出力**: 電文送信、メール送信、帳票出力、ファイル出力、DB接続
- **共通機能**: ログ出力、入力精査、認証認可、開閉局制御、業務日付管理、採番処理、コード値管理

## 2. 単体テストからシステムテスト、リグレッションテストを含めた工程全体でのテスト工数の極小化

NTF（Nablarch Testing Framework）により、自動テスト内で以下の事前条件をテストケース内でセットアップ可能：

- DBのテーブル
- HTTPセッションの内容
- バッチの入力ファイルの内容
- メッセージングキュー上の電文

また、打鍵テスト作業中の状態をテストの事前条件としてエクスポートする機能を提供し、事前条件のセットアップを簡便に行える。さらに、単体テスト工程だけでなく結合テスト・システムテストも含めた開発工程全体でのテスト工数を極小化するため、どこまでを自動テストで担保しどの部分を打鍵で確認すべきかをベストプラクティスとして体系化している。

## 3. 自前主義

OS、JDK、データベース、アプリケーションサーバ等の基盤ソフト/ミドルウェア製品を除く全ての部品をスクラッチ開発。一貫した指針のもとに実装し、アーキテクチャ一貫性の喪失・構成管理の複雑化・OSSのガバナンス問題を回避する。
