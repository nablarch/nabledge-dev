# 国際化対応

## 要求

## 要求

フレームワークが国際化対応で実装する要求の選定方針:
- 多言語化で使用頻度が高い要求に対応する
- アプリケーション毎の差異がない一般化された要求（日付、数字など）に対応する
- 一般化できない要求（名前の姓名順序・ミドルネーム、住所、通貨変換、単位変換など）は対象外

**実装される要求:**
- ユーザが言語とタイムゾーンを選択できる（リクエストパラメータまたはブラウザの言語設定）
- 選択した言語とタイムゾーンをクッキーまたはHTTPセッションで保持できる
- 指定された言語で表示できる対象:
  - 画面（メッセージ、項目名、画面名、メニュー名、ボタン名など）
  - 帳票（タイトル、項目名など）
  - 照会結果ファイル（CSVファイルのヘッダ部など）
  - メール（件名、メール本文）
  - コード（コード値としてあらかじめマスタ登録する項目の名称）
  - 日付（年月日順: YYMMDD、MMDDYY、DDMMYYなど）
  - 数字（区切り文字: 123,456.789、123.456,789など）
  - 障害メッセージ
- 指定されたタイムゾーン（サマータイム含む）に応じて日時を表示できる（基準タイムゾーンからの時差を考慮）
- リソース（ファイルやデータなど）追加のみで対応言語を追加できる

非対応の要求はアプリケーション毎に対応する。

## 全機能共通の国際化対応実装方針

## 全機能共通の国際化対応実装方針

### RDBMS上データの保持方法

RDBMS上に各国語対応が必要なデータを配置する際、必ず言語を表すカラムを持つ。

例（メッセージテーブルのカラム定義）:

| カラム名 | データ型 | 備考 |
|---|---|---|
| MESSAGE_ID | VARCHR2(10) | メッセージのキー |
| LANG | VARCHR2(10) | 対応する言語（'EN','JA'等） |
| MESSAGE | VARCHR2(150) | メッセージ |

> **注意**: この方針はフレームワーク自体にのみ影響する。フレームワークを使用するアプリケーションはこの方針に従う必要はない。

### 言語を指定した取得方法

各国語対応機能はスレッドローカルに保持した言語を使用するため、言語を明示的に渡す必要はない。複数言語を同時表示など明示的に言語を指定する場合もサポートする。

```java
// スレッドの言語でメッセージを取得
String msg = MessageResource.get("MSG0001");
// 英語のメッセージを取得
String msg = MessageResource.get("MSG0001", "EN");
```

### フレームワーク内ログ出力メッセージの使用言語

フレームワーク内のログ出力メッセージは英語を使用し、国際化は行わない。

### ファイル入出力の文字コード

ファイル入出力を伴う機能は、設定によりファイルの文字コードを指定できる。これによりアプリケーションの国際化対応が可能となる。

## Webアプリケーションに特化した国際化対応実装方針

## Webアプリケーションに特化した国際化対応実装方針

### 言語の選択と保持

フレームワークが提供する言語選択方法: ブラウザの言語設定で指定された言語のみ。

ユーザが画面上で言語を選択する方法やログインユーザに紐づく言語の選択は実現方法が多岐にわたるため提供しない。ただし、フレームワークの拡張ポイントでアプリケーション毎に言語取得方法を実装した場合、選択した言語による表示が可能。

**言語の選択優先順位（リクエストごと）:**
1. クッキーまたはHTTPセッションに保持している言語
2. ブラウザの言語設定が取得できた場合、最も優先されている言語
3. アプリケーションで設定するデフォルト言語

選択した言語はスレッドローカルに保持され、リクエスト処理完了まで取得可能。

詳細は :ref:`ThreadContextHandler` を参照。

### タイムゾーンの選択と保持

言語の選択と保持と同じ方針。ただし、ブラウザ設定によるタイムゾーン選択は存在しない（ブラウザでタイムゾーンを指定できないため）。ユーザにタイムゾーンを選択させる場合はアプリケーション毎に実装が必要。

詳細は :ref:`ThreadContextHandler` を参照。

### JSPファイルおよび静的ファイルのパス

JSPファイルの国際化対応方法（両方サポート）:
- JSPファイル中の全文字列を国際化されたリソースから取得する方法（JSPファイルを共通化）
- JSPファイル自体を言語ごとに用意する方法

**方法の選択基準:** 多言語化対象の言語種類によりデザインが異ならない場合はJSPファイルを共通化できるため前者を選択する。デザインが異なる場合はJSPファイルを共通化できないため後者を選択する。フレームワークはどちらのケースにも対応できるよう両方の方法をサポートしている。

静的ファイルのパスを言語ごとに用意する方法（2通り）:
- 言語毎にディレクトリを分ける方法
- 言語ごとにファイルを分ける方法

JSPファイル中での国際化リソース取得: :ref:`WebView_Message` を参照。
JSPファイルの言語ごとのパス切り替え: [言語毎のコンテンツパスの切り替え](../../component/handlers/handlers-HttpResponseHandler.md) を参照。
静的ファイルの言語ごとのパス切り替え: :ref:`WebView_LangResourcePath` を参照。
