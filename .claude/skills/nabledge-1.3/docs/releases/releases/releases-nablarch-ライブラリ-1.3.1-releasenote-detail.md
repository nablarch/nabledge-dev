| 2.0 | バッチ | 常駐バッチの機能改善 | 空きスレッドが随時要求データを取得して処理するように、常駐バッチの機能改善を行いました。 これまでの常駐バッチをマルチスレッド環境下で実行した場合、処理の遅いスレッドがある場合、そのスレッドが終了するまで他のスレッドが待機します。このため、遅いスレッドが終了するまで、次のバッチ処理の実行タイミンがずれ込み、遅いスレッドの実行中に作成された要求データの取り込み遅延が発生します。この問題を解決するため、常駐バッチの機能改善を行いました。 この機能改善に伴い、常駐バッチのハンドラ構成を大きく変更しています。新規プロジェクトで常駐バッチを作成する際は、今回変更したハンドラ構成にて常駐バッチを使用してください。 なお、今回の機能改善による1.4.2以前の常駐バッチのハンドラ構成への影響はありません。1.4.2以前の常駐バッチのハンドラ構成に変更を加えることなく、常駐バッチを引き続き使用することが可能です。 | 変更 | 1.3.1 | なし | － | ■Nablarch Application Framework 解説書 バッチ実行制御基盤 ■Nablarch Application Framework 解説書 データリーダリファレンス：データベーステーブルキューリーダ |  |
