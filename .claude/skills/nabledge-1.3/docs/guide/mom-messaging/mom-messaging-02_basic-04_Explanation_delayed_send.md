# 応答不要メッセージ送信処理のアプリケーション構造

## 概要

共通アクションクラスが電文送信テーブル（一時テーブル）から未送信データを取得して電文作成・送信を行う。アプリケーション開発者が作成する成果物:

- フォーマット定義ファイル（電文レイアウト定義）
- SQLファイル（以下3種類）:
  - 未送信データ取得用SELECT文（条件に未送信ステータスを含める必要あり）
  - 送信完了後の処理済みステータス更新用UPDATE文
  - 送信失敗時のエラーステータス更新用UPDATE文
- 送信用一時テーブル
- 一時テーブルのステータス更新用Formクラス

> **注意**: Formクラスに必要なプロパティはステータス更新に必要なテーブル項目のみ。テーブルレイアウトをプロジェクト共通で定義することで単一Formクラスを全応答不要メッセージ送信処理で共用可能。詳細は :ref:`sendFormSample` 参照。

> **注意**: 電文送信テーブルへのデータ登録は、前処理（画面オンライン処理やバッチ処理）で行われる。このコンポーネントはテーブルからの読み取りと送信のみを担当し、データ登録は行わない。

## 処理の流れ

1. SQLファイルから電文送信用データ取得のためのSELECT文を取得する
2. 一時テーブルから電文送信用データを取得する
3. フォーマット定義ファイルをもとに送信用電文を生成する
4. 電文を送信する
5. 一時テーブルのステータス更新用Formクラスを生成する
6. ステータス更新用UPDATE文を取得する
7. FormクラスとUPDATE文で一時テーブルのステータスを更新する

> **注意**: ステータス更新により以下を防止できる:
> - 同一電文の複数回送信
> - エラーデータ（ポイズンデータ）の繰り返し送信による障害
