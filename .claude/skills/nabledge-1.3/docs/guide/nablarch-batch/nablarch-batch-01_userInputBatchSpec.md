# ユーザ情報入力バッチの仕様

## 機能概要

ユーザ情報ファイルを入力として、ユーザ情報テンポラリテーブルを作成する。

![ユーザ情報入力バッチ処理フロー](../../../knowledge/guide/nablarch-batch/assets/nablarch-batch-01_userInputBatchSpec/userInputBatch.jpg)

> **注意**: CSVファイルと固定長ファイルの2種類のファイル入力バッチを提供する。

**INPUT**: ユーザ情報入力ファイル  
**OUTPUT**: ユーザ情報テンポラリ

**精査仕様**

ファイルのレイアウト順: ヘッダレコード → データレコード（存在しなくてもよい）→ トレーラレコード → エンドレコード

[レイアウト精査]
1. 1レコード目はヘッダーレコードであること
2. 2レコード目以降にデータレコードが複数存在すること。データレコードが存在しない場合はトレーラレコードであること
3. データレコードの次のレコードはトレーラレコードであること
4. 最終レコードはエンドレコードであること

[トレーラレコードの精査]
1. トレーラレコードの総レコード数がデータレコードのレコード数と一致すること
