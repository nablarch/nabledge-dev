# Actionクラス内で実行するSQL文の条件網羅テストを簡単に実施する方法はありますか？

## SQL条件網羅テストへのクラス単体テスト併用

SQLの条件網羅テストはリクエスト単体テストのみで実施する必要はない。クラス単体テストを併用すること。

Actionクラス内のビジネスロジック・SQL文は入力項目増加によりテストケース数が急増する。全てをリクエスト単体テストで実施すると、条件に合致したデータのセットアップに多くの工数を消費する。

対策: 当該ロジックをコンポーネントクラスとして分離し、クラス単体テストでテスト実施する。SQL文・ビジネスロジックへの入力変数や参照テーブルの内容を直接セットアップでき、作業工数およびレビュー工数の削減が期待できる。
