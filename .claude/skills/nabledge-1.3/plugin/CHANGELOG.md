# 変更履歴

nabledge-1.3プラグインの主な変更内容を記録しています。

フォーマットは [Keep a Changelog](https://keepachangelog.com/en/1.0.0/) に基づいています。

## [1.0] - 2026-07-24

### 変更

- バージョン1.0として安定版リリースしました。全Nablarchバージョン（6、5、1.4、1.3、1.2）に対応し、知識生成パイプラインはルールベースで動作します。コア機能（Q&A、コード分析）が揃い、本番環境での利用に適した品質水準に達したことを宣言します。

## [0.4] - 2026-07-10

### 変更
- 質問回答の「参照」欄に、引用したドキュメントの名称・ファイルパス・セクション名が表示されるようになりました。参照先のドキュメントとセクションをすぐに確認できます。
- コード分析の「詳細」リンクに、参照先のドキュメントの名称・ファイルパス・セクション名が表示されるようになりました。参照先のドキュメントとセクションをすぐに確認できます。

## [0.3] - 2026-06-18

### 追加
- 質問に含まれる概念（データ変換、排他制御など）から、その概念を担当するNablarchクラスが記述されたページを推論して選定できるようになりました。クラス名を知らなくても、より適切なドキュメントページが回答に使われるようになりました。

### 変更
- コード分析の依存グラフに表示するクラス数を最大15件に制限しました。大規模なコードベースでも見やすい図が生成されるようになりました。

## [0.2] - 2026-05-26

### 変更
- 知識ファイルの生成方式をAIによる要約からルールベース変換に切り替えました。AI特有のハルシネーションが発生しなくなり、ミッションクリティカルなシステムでも安心して利用できるようになりました。
- ワークフロー実行中にスクリプトの許可プロンプトが表示されなくなりました。作業の中断なくスムーズにワークフローを実行できます。
- 質問回答のフローを見直し、処理方式や目的に応じてより適切な回答ができるようになりました。
- 質問回答の検索処理をメインコンテキスト上で実行するように変更しました。これまでは別エージェントで検索を実行し結果のみを返す方式でしたが、処理方式の確認やコード分析の対象確認などユーザーへのヒアリングができませんでした。より適切な回答のためにヒアリングできる方式に切り替えています。なお、回答内容がメインコンテキストに蓄積されるようになったため、長時間利用後はセッションをクリアするか新規セッションを開始することをおすすめします。

### 修正
- 閲覧用ドキュメント内のリンクが正しく表示されない問題を修正しました。
- コード分析のシーケンス図にJavaメソッド名が正しく出力されるようになりました。実際のコードと対応した図を確認できます。
- コード分析の処理フローにヘルパー/プライベートメソッドが含まれるようになりました。より詳細な処理の流れを把握できます。
- コード分析の出力先ファイルパスが結果に明示されるようになりました。出力ファイルをすぐに確認できます。

## [0.1] - 2026-03-30

### 追加

- Nablarch 1.3の全ドキュメントをカバーする知識ファイルを追加しました。
