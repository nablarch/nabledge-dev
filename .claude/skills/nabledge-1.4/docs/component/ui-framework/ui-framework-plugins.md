# UIプラグイン

## UIプラグイン概要

UIプラグインはUI開発基盤の全成果物を管理する単位。各プラグインは単一の機能・目的に対して作成され、機能実現に必要な成果物一式（タグファイル、JavaScript、CSS、テスト/サンプルコード）を含む。

プロジェクト側でカスタマイズする場合は、対応するプラグインをコピーして必要な改修を行う（[../development_environment/modifying_code_and_testing](ui-framework-modifying_code_and_testing.md) 参照）。

| カテゴリ | 内容 | 主要プラグイン |
|---|---|---|
| 外部ライブラリ | サードパーティ製ライブラリ（プロジェクト側でのカスタマイズは不可） | jQuery（DOM API抽象化）、sugarjs（ネイティブAPI拡張・ES5ポリフィル）、requirejs（動的スクリプトロード・ミニファイ）、fontawesome（アイコン用WebFont） |
| コアJS部品 | JavaScriptで実装された共通処理とスクリプトロード定義（プロジェクト側でのカスタマイズは不可） | スクリプトロード/ドキュメントロード共通処理、Nablarch Submit機能連動カスタムイベント、JavaScriptウィジェット共通基底クラス |

## UIプラグインの構造

1つのプラグインには、機能実現に必要なリソース・ドキュメント・テストが含まれる。

ファイル構成例（UIウィジェット：カレンダー日付入力ウィジェットプラグイン）：

```bash
nablarch-widget-field-calendar    # プラグイントップ
  ├── package.json                # プラグイン依存関係定義
  ├── bin                         # 開発用コマンド
  ├── ui_local                    # ローカル表示/開発用コマンドソースコード
  │    └── js
  ├── ui_public                   # サーバ動作成果物(サーブレットコンテキスト配下の配置物)
  │    ├── css
  │    │     └── ui
  │    │            └── field-calendar.less
  │    ├── img
  │    ├── include
  │    ├── js
  │    │     └── nablarch
  │    │           └── ui
  │    │                 ├── DatePicker.js
  │    │                 └── DatePicker.template
  │    └── WEB-INF
  │          └── tags
  │                └── widget
  │                      └── field
  │                            └── calendar.tag
  └── ui_test                     # テスト/動作デモ用ソースコード
```

```

```

## UIプラグインのバージョンについて

| 区分 | 意味 | 初期値 |
|---|---|---|
| メジャーバージョン | 後方互換性を持たない修正に対してインクリメント。後方互換性をもたない修正は通常、個別の派生プラグインとして開発されるため、実際の運用では **1** 固定。 | 1 |
| マイナーバージョン | 後方互換性を保つ修正（基本的には機能追加）に対してインクリメント | 0 |
| パッチ番号 | 後方互換性を保つ不具合の修正に対してインクリメント | 0 |
