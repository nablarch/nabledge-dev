# JSPカスタムタグライブラリ

## 概要

Webアプリケーションの画面作成を助けるカスタムタグを提供する。

**制約**:
- JSP2.1以降をサポートしているWebコンテナで動作する
- カスタムタグはXHTML 1.0 Transitionalに対応した属性をサポートする
- クライアントのJavaScriptが必須である

本機能は入力画面→確認画面→完了画面といった画面遷移によるWebアプリケーションを対象としており、リッチクライアント(Ajax、Flashなど)には対応していない。

**想定する画面処理方式**:
- 画面遷移は全てPOST(リンク含む)を使用する
- アクションはPOSTを受けてJSPにフォワードする。JSPへはアクション経由でのみアクセス可とする
- アクションから別アクションへはリダイレクトではなくフォワードを使用する
- JSPはカスタムタグを使用して作成する
- 入力データはバリデーション機能を使用してエンティティ(又は独自クラス)に設定する
- 入力データのリクエストパラメータ名: オブジェクトとMap型はドット区切り、List型は角括弧(インデックス付き)
- 画面を跨いで使用するデータはhiddenで保持する(セッション不使用)
- DBコミットを伴う処理画面(確認画面など)は二重サブミットを防止する

> **注意**: HTML5で追加された属性のうち、対応するのは以下のみ。
> - autocomplete (input、password、form)
> - autofocus (input、textarea、select、button)
> - placeholder (text、password、textarea)
> - maxlength (textarea)
> - multiple (input)

## 特徴

**生産性の向上**: 入力画面と確認画面が1対1に対応する場合、JSPを共通化する機能を提供する。共通化により重複記述を最小化できる。

**画面設計の柔軟性**:
- 入力データの画面間持ち回りはクライアント側(ブラウザ)で保持するため、ブラウザの「戻るボタン」や「複数ウィンドウ」をアプリケーションで使用できる
- 二重サブミット防止: JavaScriptによるクライアント側と、一意なトークンによるサーバ側の両方で防止する機能を提供する

## 要求

**実装済み**:
- フォームの複数ボタン/リンクから、それぞれ異なるURIにサブミットできる
- フォームの複数ボタン/リンク毎にパラメータを変更できる
- ボタンの二度押しによるエラー発生やデータ不整合を防止できる
- 入力画面の再表示時(精査エラーや確認画面から戻る場合など)に、入力データを復元した状態で表示できる
- ブラウザの戻るボタン使用時にキャッシュの使用を抑制できる
- HTMLエスケープ処理ができる
- エラー発生時に全エラー内容のメッセージを画面に表示できる
- 入力値チェックエラー時にエラー原因となった入力項目をハイライト表示できる
- エラーメッセージをレベル毎に色分け(Error:赤、Warning:青、Info:緑など)して表示できる
- リストボックス表示を作りこみせずに実現できる
- コード値をリストボックス表示できる( [../03_Common/02_CodeManager](libraries-02_CodeManager.md) 参照)
- 入力画面と確認画面のJSPを共通化できる
- クライアント側(hidden)で保持するデータの改ざんを防止できる
- 認可判定の結果に応じて画面項目(メニューやボタンなど)の表示・非表示を切り替えられる
- 開閉局判定の結果に応じて画面項目(メニューやボタンなど)の表示・非表示を切り替えられる

**未実装**:
- 確認画面での変更箇所の色付け表示
- JavaScriptに埋め込む動的な値へのエスケープ処理

**未検討**(対応方針未定):
- システム利用ユーザ間で共有したい情報の表示(例: 「今日は早帰り日です」)
- 実行環境の表示(例: 単体テスト環境かST環境かを画面表示)
- 表示項目の内容に応じた表示形式の制御(例: 金額項目の右寄せ折り返し表示)
- ヘッダーやフッターなど全画面共通部分のレイアウト定義
- 精査エラー発生時に先頭のエラー項目へのフォーカス
- エンターキーによる入力項目間の移動
- エンターキーによるサブミットの許可・不許可制御
- キーバインドのカスタマイズ(例: F1キーでサブミット)
- クライアントサイドのバリデーション
- ラジオボタンのクリア(未選択状態への変更)
- 画面全体の背景色の指定
- 各業務で指定がない場合の初期フォーカスのデフォルト指定(例: 一番上の入力ボックス)
- 項目名や見出し、入力ボックスなどの背景色・文字(サイズ、色、パディング)の指定
- ボタンのサイズ・色などの指定(標準ボタンの提供)

**取り下げ**:
- 複数エラー発生時にレイアウトを崩さずに全エラー内容を表示する機能: 全てのエラー内容をリスト表示する機能は提供している。ただし、「レイアウトを崩さずに」という部分については、アプリケーション開発時にレイアウトが決定するため、フレームワークとしてレイアウト崩れを防ぐ方法が提供できない。
