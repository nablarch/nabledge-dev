# 携帯端末(フィーチャーフォン)向け画面を実装する際の留意点 — 本項で説明する内容

本項では、携帯端末 (特にフィーチャーフォンと呼ばれる端末) を対象とした画面を作成する
際に留意すべき点について解説する。
これらの端末では、サブウィンドウやjavascriptの利用が制限されている機種が多く、それにより、
これまでの各節でのべたNablarchが提供する機能について、一部制約が発生する。

> **Note:**
> なお、iPhone/Android 等のいわゆるスマートフォンとよばれる端末については、
> 上記の制約が当てはまらないため、本項の説明の対象に含まない。

## 携帯端末でサポートされるNablarchの機能

以下の表は、Nablarchが提供する画面向けの機能について、携帯端末でのサポート状況をまとめたものである。

| 機能名 | サポート | 備考 |
|---|---|---|
| セッションスコープ | ○ | Cookie非対応端末では、ＡＰサーバ側でURLリライトによる セッショントラッキングが有効となっている必要がある |
| リクエストスコープ | ○ |  |
| ウィンドウスコープ | △ | リンク(**<n:a>** タグ)による遷移ではウィンドウスコープは 引き継がれない。  > **Note:** > 以下の理由から、ウィンドウスコープ内のデータ量は > 極力抑える必要がある。  > 1. >   端末の処理能力、ネットワーク帯域が限られている端末では、 >   ページサイズが大きいとユーザビリティが極端に低下する。 > 2. >   ページサイズ上限が厳しい端末ではそもそもページが >   表示できない可能性がある。 >   (2004～2005ごろのエントリモデル端末では >   20Kbyte～100Kbyte程度が上限) |
| 2重サブミット防止 | △ | 下記の理由により、クライアント側での制御は行わず、サーバサイド側で の制御(トークン制御)のみ行う。  ほとんどの携帯端末では、サブミットボタンを押下した際に、読み込み中 ダイアログボックスが表示されるUIを採用しているため、 デスクトップブラウザで発生するボタン二度押しのオペレーションミスは 発生しづらい。このため、クライアント側の二度押し制御は不要である。 |
| 送信URI動的切替 | ○ | サーバサイド側でURIを書き換えるしくみにより利用できる。 ただし、サーブレットコンテキスト外部のパスや、 外部サイトへのアクセスについては制御できない。 |
| キャッシュ防止 | ○ |  |
| ボタン・リンクによるフォーム送信 | × | javascriptが動作しない端末では実現不可能なため、サポートしない。 通常の **<n:a>** とクエリパラメータを使用すること。  > **Note:** > 携帯端末では、デスクトップブラウザに比べURIの文字列長の > 上限が厳しいので留意すること。 > (特に、docomoの端末では、512byte が上限と非常に小さい)  > このため、クエリパラメータで送信する内容は原則として固定長、 > 固定件数のデータのみとすること。  > **Note:** > **<n:button>** 、 **<n:submitLink>** を使用していた場合は、 > JSPエラーとなる。 |
| ファイルダウンロード | △ | 携帯端末でのダウンロード機能は、キャリアおよび機種に強く依存する ため、通常のリンクタグ(**<n:a>**) もしくは、 **<object>** タグを 直接使用して実装すること。  なお、いずれの場合もGETリクエストが送信されるため、 フォームパラメータの送信および、ウィンドウスコープの維持は行われない。  > **Note:** > **<n:downloadLink>** 、 **<n:downloadButton>** を使用していた場合は、 > JSPエラーとなる。 |
| ファイルアップロード | △ | 通常のアップロードタグ (<input:type='file' />) を使用する。  > **Note:** > 携帯端末でファイルアップロードがサポートされている機種は > ごく一部に限られるため、 > 通常は、メール添付による擬似アップロードなど、汎用的に利用できる > 代替手段を使用するべきである。 |
| 複数ウィンドウの制御 | × | 端末側に該当する機能が無いため、サポートしない。  > **Note:** > **<n:popupLink>** 等のタグを使用していた場合は、JSPエラーとなる。 |
| 入力画面・確認画面の共用化 | ○ |  |
| 認可・開閉局判定に基づく表示制御 | ○ |  |

## 携帯端末で使用できないカスタムタグライブラリ

Nablarchが提供する以下のカスタムタグライブラリは、上記の機能制約により正常に動作しない。
そのため、これらのタグが使用されていた場合には実行時例外が送出され、明示的なエラーとなる。

```jsp
<n:button>
<n:submitLink>
<n:pupupSubmit>
<n:popupLink>
<n:popupButton>
<n:downloadLink>
<n:downloadButton>
```
