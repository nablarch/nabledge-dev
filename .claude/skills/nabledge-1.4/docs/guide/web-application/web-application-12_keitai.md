# 携帯端末(フィーチャーフォン)向け画面を実装する際の留意点

## 携帯端末でサポートされるNablarchの機能

> **注意**: スマートフォン（iPhone/Android等）は本項の対象外。

携帯端末（フィーチャーフォン）でのNablarch機能サポート状況：

| 機能名 | サポート | 備考 |
|---|---|---|
| セッションスコープ | ○ | Cookie非対応端末ではAPサーバ側でURLリライトによるセッショントラッキングが有効になっている必要がある |
| リクエストスコープ | ○ | |
| ウィンドウスコープ | △ | `<n:a>`タグによるリンク遷移ではウィンドウスコープは引き継がれない。端末の処理能力・帯域制限およびページサイズ上限の問題から、ウィンドウスコープ内のデータ量は極力抑える必要がある（2004～2005ごろのエントリモデル端末では20Kbyte～100Kbyte程度が上限） |
| 2重サブミット防止 | △ | クライアント側制御は行わず、サーバサイドのトークン制御のみ行う。ほとんどの携帯端末ではサブミットボタン押下時に読み込み中ダイアログボックスが表示されるUIを採用しているため、デスクトップブラウザで発生するボタン二度押しのオペレーションミスは発生しづらく、クライアント側の二度押し制御は不要 |
| 送信URI動的切替 | ○ | サーブレットコンテキスト外部のパスや外部サイトへのアクセスは制御できない |
| キャッシュ防止 | ○ | |
| ボタン・リンクによるフォーム送信 | × | javascript非対応のため実現不可。`<n:a>`タグとクエリパラメータを使用すること。`<n:button>`・`<n:submitLink>`使用時はJSPエラー |
| ファイルダウンロード | △ | `<n:a>`または`<object>`タグを直接使用すること。GETリクエストのみ送信されるため、フォームパラメータ送信とウィンドウスコープ維持は不可。`<n:downloadLink>`・`<n:downloadButton>`使用時はJSPエラー |
| ファイルアップロード | △ | 通常の`<input type="file">`を使用。サポート端末が限られるため、メール添付等の代替手段を推奨 |
| 複数ウィンドウの制御 | × | サポートしない。`<n:popupLink>`等使用時はJSPエラー |
| 入力画面・確認画面の共用化 | ○ | |
| 認可・開閉局判定に基づく表示制御 | ○ | |

> **注意**: ボタン・リンクによるフォーム送信でクエリパラメータを使う場合、携帯端末はURIの文字列長上限が厳しい（docomo端末は512byte）。クエリパラメータで送信する内容は固定長・固定件数のデータのみとすること。

## 携帯端末で使用できないカスタムタグライブラリ

以下のカスタムタグライブラリは携帯端末では正常に動作せず、使用した場合は実行時例外が送出される：

```jsp
<n:button>
<n:submitLink>
<n:pupupSubmit>
<n:popupLink>
<n:popupButton>
<n:downloadLink>
<n:downloadButton>
```
