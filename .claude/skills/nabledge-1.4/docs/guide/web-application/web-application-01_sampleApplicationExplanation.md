# サンプルアプリケーションの概要

## 画面遷移

サンプルアプリケーションは次の7つの機能を持つ:
- ログイン機能
- メニュー機能
- ユーザ情報照会機能
- ユーザ情報登録機能
- ユーザ情報更新機能
- ユーザ情報削除機能
- ユーザ情報一括削除機能

本書で説明する機能: ユーザ一覧照会機能、ユーザ情報登録機能、ユーザ情報更新機能（検索・新規登録・更新の典型的な処理パターンを説明）。

ログイン機能・メニュー機能は機能が限定されすぎるため取り上げない。ユーザ情報削除機能・一括削除機能は本書で取り上げる機能を利用して実装できるため取り上げない。

## 主な仕様 - ユーザ情報照会機能

**検索条件**: ログインID、漢字氏名、カナ氏名、グループ（ドロップダウン・表示ごとに最新取得）、ユーザIDロック（ドロップダウン・表示ごとに最新取得）

**精査**: 単項目精査を実施。検索条件は最低1つ必須。

精査エラーなし → 検索実行。検索結果が上限件数を超えない場合は一覧表示（ページング対応）。

**ソート**: ログインID・漢字氏名・カナ氏名でソート可能。

**詳細表示**: 一覧のログインIDリンククリックで、ログインID・漢字氏名・カナ氏名・メールアドレス・内線番号・携帯電話番号・グループ・認可単位を表示。

## 主な仕様 - ユーザ情報登録機能

**登録項目**:
- ログインID（必須）、パスワード（必須）、パスワード（確認用）（必須）
- 漢字氏名（必須）、カナ氏名（必須）、メールアドレス（必須）
- 内線番号（ビル番号・個人番号、必須）
- 携帯電話番号（市外番号・市内番号・加入番号、任意）
- グループ（ドロップダウン・表示ごとに最新取得、必須）
- 認可単位（リストビルダー・表示ごとに最新取得、任意）

**精査**:
- 単項目精査
- パスワードとパスワード（確認用）は一致必須
- 携帯電話番号は全入力か全未入力
- ログインIDは重複不可
- グループ・認可単位はシステムに登録済みであること

精査エラー → 登録画面にエラーメッセージ表示。精査OK → 確認画面表示 → 登録指示でDBに挿入。

## 主な仕様 - ユーザ情報更新機能

**初期表示項目**: ログインID、漢字氏名、カナ氏名、メールアドレス、内線番号（ビル番号・個人番号）、携帯電話番号（市外番号・市内番号・加入番号）、グループ（ドロップダウン・表示ごとに最新取得）、認可単位（リストビルダー・表示ごとに最新取得）

**更新項目**:
- 漢字氏名（必須）、カナ氏名（必須）、メールアドレス（必須）
- 内線番号（ビル番号・個人番号、必須）
- 携帯電話番号（市外番号・市内番号・加入番号、任意）
- グループ（ドロップダウン・表示ごとに最新取得、必須）
- 認可単位（リストビルダー・表示ごとに最新取得、任意）

**精査**:
- 単項目精査
- 携帯電話番号は全入力か全未入力
- グループ・認可単位はシステムに登録済みであること

精査エラー → 更新画面にエラーメッセージ表示。精査OK → 確認画面表示 → 更新指示でDBを更新。

**排他制御あり**: 排他制御エラー発生時はユーザ情報詳細画面にエラーメッセージを表示。

## サンプルアプリケーションの実行方法

1. 開発環境構築ガイドに従いログインする。
2. トップメニューで使用したい機能をクリックする。

以下の機能はユーザ情報一覧照会の結果から実行できる: ユーザ情報更新機能、ユーザ情報削除機能、ユーザ情報一括削除機能。
