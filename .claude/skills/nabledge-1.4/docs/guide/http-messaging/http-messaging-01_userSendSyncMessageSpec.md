# ユーザ情報登録サービスの仕様

## 機能概要

ユーザ情報登録画面入力からユーザ情報登録電文を作成し外部システムに送信、応答電文から採番ユーザIDを取得して完了画面に表示する。

ユーザ登録確認画面にHTTPメッセージ送信ボタンを設け、HTTPメッセージ送信ボタンに対応する処理を行うアクションにユーザ情報登録電文の送信処理を実装する。

### ユーザ登録情報送信処理

- **INPUT**: 画面で入力されたユーザ情報
- **OUTPUT**: 要求電文

外部システムからの要求電文でユーザ登録する場合、後続処理でパスワードと権限情報にデフォルト値が設定されるため、画面入力のパスワードと権限情報は使用しない。

- 要求電文・応答電文はデータレコードのみ必須
- 要求電文データレコードのレイアウト仕様: [user-send-sync-request-format](#)
- 応答電文データレコードのレイアウト仕様: [user-send-sync-response-format](#)
- HTTPレスポンスコード200の場合: 完了画面を表示
- HTTPレスポンスコード200以外の場合: 業務エラーを送出し、登録画面にエラーメッセージを表示
- 通信先エラー（接続エラー等）: システムエラーを送出しエラー画面に遷移（アーキテクトにより提供されるため業務Actionは意識不要）

## 要求電文仕様

### データレコード

| 項目名 | データタイプ | 多重度 | 説明 |
|---|---|---|---|
| データ区分 | X | 1 | 半角:1桁 |
| ログインID | X | 1 | 半角:20桁以下:必須 |
| 漢字氏名 | XN | 1 | 全角:50桁以下:必須 |
| カナ氏名 | N | 1 | 全角カナ:50桁以下:必須 |
| メールアドレス | X | 1 | 半角:100桁以下:必須 |
| 内線番号(ビル番号) | X | 1 | 半角数字:2桁以下:必須 |
| 内線番号(個人番号) | X | 1 | 半角数字:4桁以下:必須 |
| 携帯電話番号 | OB | 0..3 | 携帯電話番号 |
| フレームワーク制御ヘッダ | OB | 0..1 | フレームワーク制御項目 |

### 携帯電話番号レコード

| 項目名 | データタイプ | 多重度 | 説明 |
|---|---|---|---|
| 携帯電話番号(市外) | X | 0..1 | 半角数字:3桁以下 |
| 携帯電話番号(市内) | X | 0..1 | 半角数字:4桁以下 / ※携帯電話番号(市外)が入力された場合は必須、未入力の場合は未入力であること |
| 携帯電話番号(加入) | X | 0..1 | 半角数字:4桁以下 / ※携帯電話番号(市外)が入力された場合は必須、未入力の場合は未入力であること |

### ヘッダレコード

| 項目名 | データタイプ | 多重度 | 説明 |
|---|---|---|---|
| ユーザID | X | 0..1 | ユーザID |
| 再送要求フラグ | X9 | 0..1 | '0': 初回送信 / '1': 再送要求 / 空白: 再送不要 / デフォルト値'0'を設定 |

### HTTPレスポンスコード仕様

| No | HTTPレスポンスコード | 原因 | 備考 |
|---|---|---|---|
| 1 | 200 | 正常終了 | |
| 2 | 200以外 | ステータスコード不正 | 業務アプリケーションで判定し、業務エラーとする |
| 3 | -- | 要求電文データレコード部レイアウト不正 | 送信前にエラーとなるため、HTTPレスポンスは返却されない |
| 4 | -- | 応答電文のデータレコード部レイアウト不正 | 通信先のエラーとなるため、返却値は不明 |
| 5 | -- | 接続タイムアウトが発生した場合 | 送信前にエラーとなるため、HTTPレスポンスは返却されない |
| 6 | -- | その他の意図しないエラーにより業務処理が失敗した場合 | |
