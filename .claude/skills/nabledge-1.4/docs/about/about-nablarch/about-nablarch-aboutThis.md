# 想定読者、位置付け、対象工程、注意事項

## 想定読者

本書が想定している読者は、アプリケーションプログラマである。アプリケーションプログラマとは、次のような人を指す。
アプリケーションプログラマ：
プロジェクトにおいて、アプリケーションフレームワークを使用してプログラミングする人。設定ファイルや、アプリケーションフレームワークの機能を拡張するコンポーネントの作成は行わない。

## 本書の位置付け

本書は開発に必要な最低限の情報を記載している。より詳細な知識を得る必要が発生した場合は、開発標準の文書やアプリケーションフレーム
ワークのJavaDocを参照。

## 本書が対象とする工程

本書の対象工程は、プログラミング・単体テスト工程である。

## 注意事項

本書を読む上での注意事項を記載する。

**記載しているサンプルプログラムソースコードの注意事項**

* 説明箇所をはっきりさせるため、一部内容を省略している(～中略～などと表記)。このため、ソースコードの全量については、Nablarchの提供するサンプルアプリケーションを参照すること。
* 見易さのため改行位置を変えているところがある。
* 以下のように記述が始まるコメントは、説明のために追加したコメントである。実際にソースコードを作成するときは、それらのような詳細なコメントを記述する必要は **ない**

  * `// 【説明】`
  * `/* 【説明】`
  * `<!-- 【説明】`
  * `<%-- 【説明】`
  * `-- 【説明】`
