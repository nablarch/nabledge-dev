# 国際化対応

## 要求

## 国際化対応の要求

要求選定の方針:
- 多言語化で使用頻度が高い要求に対応
- アプリケーション毎の差異がない一般化された要求（日付、数字など）に対応
- 名前（姓名順序、ミドルネーム）、住所、通貨変換、単位変換は一般化できないため対象外

実装する要求:
- ユーザが言語とタイムゾーンを選択できる（リクエストパラメータまたはブラウザの言語設定）
- 選択した言語とタイムゾーンを保持できる（クッキーまたはHTTPセッション）
- 指定された言語で表示できる: 画面（メッセージ、項目名、画面名、メニュー名、ボタン名など）、帳票（タイトル、項目名など）、照会結果ファイル（CSVヘッダなど）、メール（件名、本文）、コード（マスタ登録済みコード値の名称）、日付（年月日順）、数字（区切り文字）、障害メッセージ
- 指定されたタイムゾーン（サマータイム含む）に応じて日時を表示できる（基準となるタイムゾーンからの時差を考慮し表示する）
- リソース（ファイルやデータなど）追加のみで対応言語を追加できる

フレームワーク未対応の要求は、多言語化を行うアプリケーション毎に対応する。

## RDBMS上データの保持方法

## RDBMS上データの保持方法

RDBMS上に各国語対応が必要なデータを配置する際、必ず言語を表すカラムを持つポリシー。

例: メッセージを保持するデータベースのカラム定義

| カラム名 | データ型 | 備考 |
|---|---|---|
| MESSAGE_ID | VARCHR2(10) | メッセージのキー |
| LANG | VARCHR2(10) | 対応する言語（'EN', 'JA'等） |
| MESSAGE | VARCHR2(150) | メッセージ |

> **注意**: この方針はフレームワーク内部のみの方針。フレームワークを使用したアプリケーションはこの方針に従う必要はない。

## 言語を指定した取得方法

## 言語を指定した取得方法

各国語対応機能はスレッドローカルに保持した言語を使用するため、言語を明示的に渡す必要はない。複数言語を同時に1つの画面内に表示するなど言語を明示的に指定するケースのために、言語を指定した呼び出しも提供する。

スレッドに紐付けられた言語でメッセージを取得:
```java
String msg = MessageResource.get("MSG0001");
```

英語のメッセージを取得:
```java
String msg = MessageResource.get("MSG0001", "EN");
```

## 言語の選択と保持（Webアプリケーション）

## 言語の選択と保持（Webアプリケーション）

フレームワークが提供する言語選択方法: ブラウザの言語設定で指定された言語を選択する方法のみ。

ユーザ操作による言語選択（画面上での選択、ログインユーザへの言語紐付け）は実現方法が多岐にわたるため提供しない。ただし、フレームワークの拡張ポイントでアプリケーション毎に言語取得方法を実装した場合は、選択した言語による表示が可能。

言語選択の優先順位（リクエストごと）:
1. クッキーまたはHTTPセッションに保持している言語
2. ブラウザの言語設定（取得できた場合、最も優先されている言語）
3. アプリケーションで設定するデフォルト言語

選択した言語はスレッドローカルに保持し、リクエストの処理完了まで取得可能。

詳細は :ref:`ThreadContextHandler` を参照。

<summary>keywords</summary>

## タイムゾーンの選択と保持（Webアプリケーション）

## タイムゾーンの選択と保持（Webアプリケーション）

タイムゾーンの選択と保持は [i18n_lang_select_keep](#s5) と同じ方針。

ブラウザの設定でタイムゾーンを指定することができないため、ブラウザの設定によるタイムゾーン選択は存在しない。ユーザにタイムゾーンを選択させる場合はアプリケーション毎に選択方法を実装する必要がある。

詳細は :ref:`ThreadContextHandler` を参照。

## JSPファイルおよび静的ファイルのパス（Webアプリケーション）

## JSPファイルおよび静的ファイルのパス（Webアプリケーション）

JSP国際化対応の2通りの方法（両方サポート）:
1. JSPファイル中の全文字列を国際化されたリソースから取得する方法（言語間でデザインが共通の場合）
2. JSPファイル自体を言語ごとに用意する方法（言語間でデザインが異なる場合）

JSPファイルから参照される静的ファイル（画像、CSSなど）のパスについても言語ごとにファイルを用意する方法をサポート。ファイルの分け方:
- 言語毎にディレクトリを分ける方法
- 言語ごとにファイルを分ける方法

参照先:
- JSPファイル中での国際化リソース取得: :ref:`WebView_Message`
- JSPファイルの言語ごとのパス切り替え: [言語毎のコンテンツパスの切り替え](../../component/handlers/handlers-HttpResponseHandler.md)
- 静的ファイルの言語ごとのパス切り替え: :ref:`WebView_LangResourcePath`
