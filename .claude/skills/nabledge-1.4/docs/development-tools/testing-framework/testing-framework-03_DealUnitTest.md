# 取引単体テストの実施方法

## 取引単体テストの実施方法

取引単体テストでは、テスト対象のアプリケーションをアプリケーションサーバにデプロイし、手動でアプリケーションを操作しテストを行う。

同期応答メッセージ送信処理がある場合はモックアップクラスを使用する（詳細は :ref:`dealUnitTest_send_sync` 参照）。

**テスト準備**:

テスト準備として以下の作業を行う。
- データベース準備（データ投入）
- アプリケーションのデプロイ
- アプリケーションサーバ起動

> **注意**: DBダンプはテスト実行前のものが必要なため、テスト実施前に取得すること。

**収集エビデンス**:
- 画面ハードコピー
- DBダンプ（テスト実行前および実行後）

> **注意**: 画面ハードコピー取得ツール、DBダンプ取得ツール等のテスト補助ツールについては現在検討中。
