# Actionクラス内で実行するSQL文の条件網羅テストを簡単に実施する方法はありますか？

> **question:**
> Actionクラス内で実行するSQL文の条件網羅テストをリクエスト単体テストで実施すると、テスト工数が増加してしまいます。
> 増加する要因としては、単一のSQLのテストを実施したいのに、そのリクエストで使用するその他のSQLの情報等を全て用意する必要がある為です。

> また、SQLの実行結果の確認方法がリクエストのアウトプットでしかないために、
> 本当にそのSQL文で該当データが存在しなかったのか？をレビュー時に確認するのが非常に大変です。

> ※実際には、別のSQLや共通コンポーネントの結果がなかったために、たまたま同じ結果となっているだけの場合もあるため、
> 全データを確認する必要があり、レビュー工数も増加してしまいます。

> **answer:**
> 全てのテストケースをリクエスト単体テストで実施する必要はありません。
> 必要に応じてクラス単体テストを併用するようにしてください。

> ご指摘のように、Actionクラス内で実行するビジネスロジックやSQL文は、
> 入力項目数の増加に伴ってテストケース数が飛躍的に増加します。
> それら全てをリクエスト単体テストとして実施した場合、
> 条件に合致したデータをセットアップすることに多くの工数を消費してしまいます。

> そのような場合は、当該部分のロジックをコンポーネントクラスとして分離し、
> クラス単体テストによってテストを実施するようにしてください。

> これにより、SQL文やビジネスロジックに対する入力変数や、参照するテーブルの内容を直接セットアップすることになるため、
> 作業工数及びレビュー工数の削減が期待できます。
