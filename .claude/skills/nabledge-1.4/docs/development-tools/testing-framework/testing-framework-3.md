## Actionクラス内で実行するSQL文の条件網羅テストを簡単に実施する方法はありますか？

## Actionクラス内で実行するSQL文の条件網羅テストを簡単に実施する方法はありますか？

全てのテストケースをリクエスト単体テストで実施する必要はない。必要に応じてクラス単体テストを併用すること。

Actionクラス内のSQL文やビジネスロジックに対する条件網羅テストは、入力項目数の増加に伴ってテストケース数が飛躍的に増加する。これら全てをリクエスト単体テストとして実施すると、条件に合致したデータのセットアップに多くの工数を消費する。

当該ロジックをコンポーネントクラスとして分離し、クラス単体テストでテストを実施すること。これにより、SQL文やビジネスロジックへの入力変数および参照テーブルの内容を直接セットアップできるため、テスト工数とレビュー工数の削減が期待できる。
