# 国際化対応

## 要求

国際化対応で実装する要求:
- ユーザが言語とタイムゾーンを選択できる（リクエストパラメータ、ブラウザ言語設定）
- 選択した言語・タイムゾーンをクッキーまたはHTTPセッションで保持できる
- 指定された言語で表示できる対象: 画面（メッセージ/項目名/画面名/メニュー名/ボタン名）、帳票（タイトル/項目名）、照会結果ファイル（CSVヘッダ等）、メール（件名/本文）、コード（性別/単位/区分等のマスタ登録項目名称）、日付（年月日順: YYMMDD/MMDDYY/DDMMYY等）、数字（区切り文字: 123,456.789 または 123.456,789）、障害メッセージ
- 指定タイムゾーン（サマータイム含む）に応じて日時表示できる（基準タイムゾーンからの時差考慮）
- リソース（ファイル・データ）追加のみで対応言語を追加できる

フレームワークが対応しない要求（アプリケーション毎に対応）: 名前（姓名順序・ミドルネーム）、住所、通貨変換、単位変換（長さ・重さ等）

## RDBMS上データの保持方法

> **適用範囲**: 本フレームワークの全ての機能に共通する実装方針（Webアプリケーション・バッチ・その他全モジュール対象）

RDBMSに各国語対応が必要なデータを配置する際、必ず言語を表すカラムをデータベースに持つポリシー。

例: メッセージ保持テーブルのカラム定義

| カラム名 | データ型 | 備考 |
|---|---|---|
| MESSAGE_ID | VARCHR2(10) | メッセージのキー |
| LANG | VARCHR2(10) | 対応する言語('EN','JA'等) |
| MESSAGE | VARCHR2(150) | メッセージ |

> **注意**: この方針はフレームワークのみに影響する方針であり、フレームワークを使用したアプリケーションではこの方針に従う必要はない。

## 言語を指定した取得方法

> **適用範囲**: 本フレームワークの全ての機能に共通する実装方針（Webアプリケーション・バッチ・その他全モジュール対象）

各国語対応機能はスレッドローカルに保持した言語を使用するため、言語を明示的に渡す必要はない。複数言語を同時に1つの画面内に表示するケースなどに備え、言語を明示的に指定した呼び出しができるよう設計する。

スレッドに紐付けられた言語でメッセージを取得:
```java
String msg = MessageResource.get("MSG0001");
```

英語のメッセージを取得:
```java
String msg = MessageResource.get("MSG0001", "EN");
```

## 言語の選択と保持

> **適用範囲**: Webアプリケーションに特化した実装方針（バッチ等の非Web機能には適用されない）

フレームワークが提供する言語選択方法: ブラウザの言語設定で指定された言語のみ。

ユーザ操作による言語選択（画面上での選択、ログインユーザに紐づく言語選択）はフレームワークでは提供しない（実現方法・選択方法が多岐に渡るため）。ただし、拡張ポイントでアプリ毎に言語取得方法を実装した場合、選択した言語による表示が可能。

リクエストごとの言語選択優先順位:
1. クッキーまたはHTTPセッションに保持している言語
2. ブラウザの言語設定で取得できた場合、最も優先されている言語
3. アプリケーションで設定するデフォルト言語

選択した言語はスレッドローカルに保持し、リクエスト処理完了までいつでも取得できる。

詳細は :ref:`ThreadContextHandler` を参照。

## タイムゾーンの選択と保持

> **適用範囲**: Webアプリケーションに特化した実装方針（バッチ等の非Web機能には適用されない）

タイムゾーンの選択と保持は [i18n_lang_select_keep](#s5) と同じ方針。

ブラウザの設定でタイムゾーンを指定できないため、ブラウザ設定によるタイムゾーン選択は存在しない。ユーザにタイムゾーンを選択させる場合はアプリケーション毎に選択方法を実装する必要がある。

詳細は :ref:`ThreadContextHandler` を参照。

## JSPファイルおよび静的ファイルのパス

> **適用範囲**: Webアプリケーションに特化した実装方針（バッチ等の非Web機能には適用されない）

JSPファイルの国際化対応方法（両方サポート）:
1. JSPファイル中に記述する全ての文字列を国際化されたリソースから取得するように記述する方法
2. JSPファイル自体を言語ごとに用意する方法

両方の方法をサポートする理由: 多言語化対象の言語種類により、デザインが異ならないためJSPファイルを共通化できる場合と、デザインが異なるためJSPファイルを共通化できない場合があり、どちらのケースにも対応するためである。

言語ごとにファイルを用意する場合の方法:
- 言語毎にディレクトリを分ける方法
- 言語ごとにファイルを分ける方法

- JSPファイル中での国際化リソース取得: :ref:`WebView_Message`
- JSPファイルの言語ごとのパス切り替え: [言語毎のコンテンツパスの切り替え](../../component/handlers/handlers-HttpResponseHandler.md)
- 静的ファイルの言語ごとのパス切り替え: :ref:`WebView_LangResourcePath`
