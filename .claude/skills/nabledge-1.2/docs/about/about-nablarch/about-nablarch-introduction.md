# 序 : Nablarch Application Framework (NAF)とは ?

## NAFの設計思想

NAF (Nablarch Application Framework) はJavaベースの基幹システム向けアプリケーションフレームワーク。3つの設計思想を基本とする。

## 1. エンタープライズシステムに必要な機能要素を包括的に提供

NAFが提供する機能：
- **アプリケーション実行機能**: 画面オンライン入力、バッチ入力、他システムからの要求電文
- **外部出力機能**: 電文送信、メール送信、帳票出力、ファイル出力、DB接続
- **共通機能**: ログ出力、入力精査、認証認可、開閉局制御、業務日付管理、採番処理、コード値管理

## 2. テスト工数の極小化

NAFと並行開発されたNTF (Nablarch Testing Framework) により以下を実現：
- DBテーブル、HTTPセッション、バッチ入力ファイル、メッセージングキュー上の電文など、業務機能実行に必要なすべての事前条件をテストケース内でセットアップ可能
- 打鍵テスト作業中の状態をテストの事前条件としてエクスポートするなど、事前条件のセットアップを簡便に行える機能を提供
- 単体テスト工程だけでなく結合テスト・システムテストを含む開発工程全体で、どこまでを自動テストで担保しどの部分を打鍵で確認すべきかをベストプラクティスとして体系化し、テスト工数を極小化

## 3. 自前主義

NAFはOS・JDK・データベース・アプリケーションサーバ等の基盤ソフト/ミドルウェアを除き、全部品をスクラッチ開発。理由：
- 複数プロダクト混在によるアーキテクチャ一貫性の喪失を回避
- 各プロジェクトの独立した運営方針に起因する長期的な構成管理の複雑化を回避
- OSSではコードの出所管理が実質不可能であり、ソースコードガバナンス上の問題（特に金融系基幹システム開発）を回避
