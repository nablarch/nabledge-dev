# ユーザ情報入力バッチの仕様

## 機能概要

ユーザ情報ファイルを入力として、ユーザ情報テンポラリテーブルを作成する。

![ユーザ情報入力バッチ処理フロー](../../../knowledge/guide/nablarch-batch/assets/nablarch-batch-01_userInputBatchSpec/userInputBatch.jpg)

> **注意**: CSVファイルと固定長ファイルの２種類のファイル入力バッチを提供する。

## ユーザ情報入力処理

- **INPUT**: ユーザ情報入力ファイル
- **OUTPUT**: ユーザ情報テンポラリ

### レイアウト精査仕様

ファイルのレイアウトは、ヘッダレコード→データレコード(データレコードは、存在していなくても良い)→データレコード→エンドレコードの順であること。

1. 1レコード目はヘッダーレコードであること。
2. 2レコード目以降にデータレコードが複数存在していること。ただし、データレコードが存在しない場合は、トレーラレコードであること。
3. データレコードの次のレコードは、トレーラレコードであること。
4. 最終レコードはエンドレコードであること。

### トレーラレコード精査

トレーラレコードの総レコード数が、データレコードのレコード数と一致すること。
