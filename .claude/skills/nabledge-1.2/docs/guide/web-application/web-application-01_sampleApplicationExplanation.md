## 主な仕様 - ユーザ情報照会機能

検索条件:
- ログインID
- 漢字氏名
- カナ氏名
- グループ（ドロップダウンリスト、表示のたびに最新情報を取得）
- ユーザIDロック（ドロップダウンリスト、表示のたびに最新情報を取得）

精査:
- 単項目精査
- 検索条件は最低1つ必須

検索結果:
- 検索上限件数以下の場合、ユーザ情報一覧照会画面に表示
- 1ページ表示上限超過時はページング機能でページ遷移
- ソート可能項目: ログインID、漢字氏名、カナ氏名

詳細表示（ログインIDリンクから遷移）: ログインID、漢字氏名、カナ氏名、メールアドレス、内線番号、携帯電話番号、グループ、認可単位

## 主な仕様 - ユーザ情報登録機能

登録フィールド:
- ログインID（必須）
- パスワード（必須）
- パスワード（確認用）（必須）
- 漢字氏名（必須）
- カナ氏名（必須）
- メールアドレス（必須）
- 内線番号（ビル番号・個人番号、必須）
- 携帯電話番号（市外番号・市内番号・加入番号、任意）
- グループ（ドロップダウンリスト、表示のたびに最新取得、必須）
- 認可単位（ドロップダウンリスト、表示のたびに最新取得、任意）

精査:
- 単項目精査
- パスワードとパスワード（確認用）は一致必須
- 携帯電話番号は全入力または全未入力
- ログインIDは既登録のものは使用不可
- グループ・認可単位はシステムに登録済みであること

フロー: 精査エラー時は登録画面にエラー表示。精査成功時は確認画面を表示し、登録指示でDBに登録。

## 主な仕様 - ユーザ情報更新機能

初期表示フィールド: ログインID、漢字氏名、カナ氏名、メールアドレス、内線番号（ビル番号・個人番号）、携帯電話番号（市外番号・市内番号・加入番号）、グループ（ドロップダウンリスト）、認可単位（ドロップダウンリスト）

更新フィールド:
- 漢字氏名（必須）
- カナ氏名（必須）
- メールアドレス（必須）
- 内線番号（ビル番号・個人番号、必須）
- 携帯電話番号（市外番号・市内番号・加入番号、任意）
- グループ（ドロップダウンリスト、表示のたびに最新取得、必須）
- 認可単位（ドロップダウンリスト、表示のたびに最新取得、任意）

精査:
- 単項目精査
- 携帯電話番号は全入力または全未入力
- グループ・認可単位はシステムに登録済みであること

フロー: 精査エラー時は更新画面にエラー表示。精査成功時は確認画面を表示し、更新指示でDB更新。排他制御エラー時はユーザ情報詳細画面にエラー表示。

## サンプルアプリケーションの実行方法

実行手順:
a) 開発環境構築ガイドに従いログインする。
b) トップメニューで使用したい機能をクリックする。

以下の機能は、ユーザ情報一覧照会の結果から実行できる:
- ユーザ情報更新機能
- ユーザ情報削除機能
- ユーザ情報一括削除機能
