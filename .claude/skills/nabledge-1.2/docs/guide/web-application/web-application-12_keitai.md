# 携帯端末(フィーチャーフォン)向け画面を実装する際の留意点

## 携帯端末でサポートされるNablarchの機能

> **注意**: iPhone/AndroidなどのスマートフォンはJavascript・サブウィンドウへの制約が当てはまらないため、本項の対象外である。

携帯端末（フィーチャーフォン）向けNablarch機能のサポート状況：

| 機能名 | サポート | 備考 |
|---|---|---|
| セッションスコープ | ○ | Cookie非対応端末では、APサーバ側でURLリライトによるセッショントラッキングを有効にする必要がある |
| リクエストスコープ | ○ | |
| ウィンドウスコープ | △ | `<n:a>` タグによるリンク遷移ではウィンドウスコープは引き継がれない。端末の処理能力・ネットワーク帯域の制限やページサイズ上限（2004〜2005ごろのエントリモデル端末では20Kbyte〜100Kbyte程度が上限）のため、スコープ内データ量は極力抑えること |
| 2重サブミット防止 | △ | クライアント側制御は行わず、サーバサイドのトークン制御のみ使用する。ほとんどの携帯端末ではサブミットボタン押下時に読み込み中ダイアログボックスが表示されるUIを採用しているため、デスクトップブラウザで発生するボタン二度押しのオペレーションミスは発生しづらく、クライアント側の二度押し制御は不要である |
| 送信URI動的切替 | ○ | サーブレットコンテキスト外部パスや外部サイトへのアクセスは制御不可 |
| キャッシュ防止 | ○ | |
| ボタン・リンクによるフォーム送信 | × | Javascript非動作端末では実現不可。通常の `<n:a>` とクエリパラメータを使用すること。`<n:button>` / `<n:submitLink>` を使用するとJSPエラーとなる |
| ファイルダウンロード | △ | `<n:a>` または `<object>` タグを直接使用すること。GETリクエストのためフォームパラメータ送信・ウィンドウスコープの維持は不可。`<n:downloadLink>` / `<n:downloadButton>` はJSPエラーとなる |
| ファイルアップロード | △ | 通常の `<input type='file'>` を使用。対応機種が少ないため、メール添付等の汎用的な代替手段を使用すること |
| 複数ウィンドウの制御 | × | 端末に該当機能が無いため非サポート。`<n:popupLink>` 等はJSPエラーとなる |
| 入力画面・確認画面の共用化 | ○ | |
| 認可・開閉局判定に基づく表示制御 | ○ | |

> **注意**: 携帯端末はデスクトップブラウザに比べURIの文字列長上限が厳しい。特にdocomo端末は512byteが上限のため、クエリパラメータで送信するデータは原則として固定長・固定件数のみとすること。

## 携帯端末で使用できないカスタムタグライブラリ

携帯端末では機能制約により以下のカスタムタグは正常に動作せず、使用すると実行時例外（JSPエラー）が送出される：

- `<n:button>`
- `<n:submitLink>`
- `<n:pupupSubmit>`
- `<n:popupLink>`
- `<n:popupButton>`
- `<n:downloadLink>`
- `<n:downloadButton>`
