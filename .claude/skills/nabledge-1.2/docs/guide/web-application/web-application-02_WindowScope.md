# Appendix A: ウィンドウスコープ概要

## ウィンドウスコープ概要

ウィンドウスコープは、リクエストを跨るデータを格納する領域。セッションスコープがサーバ上に格納するのに対し、ウィンドウスコープは複数画面間をhiddenタグで持ちまわることで実現される。複数ウィンドウを並列動作させることが可能で、ブラウザのヒストリバックによる遷移も可能。

> **注意**: ウィンドウスコープに格納された変数は、カスタムタグを使用することで自動的にhiddenタグに変換される。**アプリケーションプログラマがJSPにhiddenタグを書く必要はない。**

## セッションスコープとの使い分け

基本的に、**ウィンドウスコープを使用**する（セッションスコープではなく）。

- **ウィンドウスコープを使用する場合**: ウィンドウ間で共有する必要のないデータ（ほとんどの場合）
- **セッションスコープを使用する場合**: ウィンドウ間でデータを共有する必要がある場合（例: ショッピングカートの商品情報など、複数ウィンドウを使用していてもユーザに対して1つにしなければならないデータ）

ウィンドウスコープはウィンドウ内のデータ領域であるため、ウィンドウ間のデータ共有には使用できない。
