# 業務アプリケーション開発手順

## 開発手順の目的と概要

NablarchウェブアプリケーションフレームワークでWebアプリケーションを開発する手順を説明するガイド。バッチ等も開発プロセスは大きく変わらないため、代表として画面オンライン処理を取り上げる。

**目的**: アプリケーションプログラマが、Nablarch Application Frameworkを使用した画面オンライン処理の開発手順を理解すること。

## 対象工程と前提条件

**対象工程**: プログラミング・単体テスト工程のみが対象。

**前提条件**: 本ガイド記載の開発手順では以下の前提条件を置いている。
- 外部設計書、内部設計書が完成していること。

## サンプルアプリケーションを使った開発手順の例

Nablarchのサンプルアプリケーションへ更新機能を追加する実装を例として、画面オンライン処理の開発手順を説明する。

**注意**: 更新機能はサンプルアプリケーションに実装済みである。以下では、簡易的な更新機能を例として取り上げる。

開発手順の各ステップ:
1. 仕様確認 (01_spec)
2. 処理フロー設計 (02_flow)
3. データセットアップ (03_datasetup)
4. エンティティ作成 (04_create_entity)
5. フォーム作成 (05_create_form)
6. 初期表示画面 (06_initial_view)
7. 確認画面 (07_confirm_view)
8. 完了処理 (08_complete)
9. 確認操作 (09_confirm_operation)
