# ユーザ情報登録サービスの仕様

## 機能概要

外部システムからの送信電文を受信し、ユーザ情報テンポラリテーブルにレコードを登録するサービス。

**処理フロー**:
1. 外部システムからユーザ情報登録電文を受信
2. データレコードの情報をユーザ情報テンポラリに登録
3. 処理結果を応答電文として返却

ユーザ情報テンポラリテーブルは、後続処理（常駐バッチ）によって一定時間間隔で監視され、未処理データが存在する場合にユーザ情報を登録する。常駐バッチの実装方法は [../../04_Explanation_batch/index](../nablarch-batch/nablarch-batch-04_Explanation_batch.md) を参照。

**主な仕様（ユーザ情報登録処理）**:
- INPUT: ユーザ情報登録電文
- OUTPUT: ユーザ情報テンポラリ、応答電文

**精査仕様**:
- 要求電文レイアウト: フレームワーク制御ヘッダ（ヘッダレコード）→ ユーザ情報データレコードの順であること
- ヘッダレコード・データレコードのレイアウト仕様・ドメイン精査は要求電文仕様を参照
