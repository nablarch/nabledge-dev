# JSPカスタムタグライブラリ

## 概要

**制約**:
- JSP2.1以降をサポートしているWebコンテナが必要
- カスタムタグはXHTML 1.0 Transitional対応属性のみサポート
- クライアントのJavaScriptが必須

**推奨画面処理方式（本機能が最も効果的に機能する前提）**:
- 画面遷移は全てPOST（リンク含む）
- アクションはPOSTを受けてJSPにフォワード。JSPへはアクション経由でのみアクセス可
- アクションから別アクションへの直接呼び出しはリダイレクトではなくフォワードを使用
- JSPはカスタムタグを使用して作成
- 入力データはバリデーション機能でエンティティ（又は独自クラス）に設定
- リクエストパラメータ名: オブジェクト・Map型はドット区切り、List型は角括弧＋インデックス
- 画面跨ぎデータはクライアント側（hidden）で保持（セッション不使用）
- コミットを伴う画面（確認画面など）は二重サブミットを防止

**非対応**: リッチクライアント（Ajax、Flash等）には対応していない。入力画面→確認画面→完了画面の画面遷移型Webアプリケーションが対象。

> **注意**: HTML5で追加されたタグと属性については、以下の属性のみ対応する。
> - autocomplete (input、password、form)
> - autofocus (input、textarea、select、button)
> - placeholder (text、password、textarea)
> - maxlength (textarea)
> - multiple (input)

## 特徴

**生産性の向上**: 入力画面と確認画面が1対1対応する場合にJSPを共通化する機能を提供。重複記述を最小化可能。

**画面設計の柔軟性**:
- 入力データをクライアント側（ブラウザ）に保持するため、ブラウザの戻るボタンや複数ウィンドウが使用可能
- 二重サブミット防止: JavaScriptによるクライアント側 + 一意なトークンによるサーバ側の両方で防止

## 要求

**実装済み**:
- 複数ボタン/リンクからそれぞれ異なるURIへのサブミット
- 複数ボタン/リンク毎のパラメータ変更
- ボタン二度押しによるエラー・データ不整合防止
- 入力画面再表示時の入力データ復元（精査エラー・確認画面から戻る場合など）
- ブラウザの戻るボタン使用時のキャッシュ使用抑制
- HTMLエスケープ処理
- エラーメッセージの全件表示
- バリデーションエラー発生時の入力項目ハイライト表示
- エラーメッセージのレベル毎の色分け表示（Error:赤、Warning:青、Info:緑など）
- リストボックス表示（作りこみ不要）
- コード値のリストボックス表示
- 入力画面と確認画面のJSP共通化（重複記述抑制）
- クライアント側（hidden）保持データの改ざん防止
- リストデータ表示のページング（作りこみ不要）
- 認可判定結果に応じた画面項目の表示・非表示切り替え
- 開閉局判定結果に応じた画面項目の表示・非表示切り替え

**未実装**:
- 確認画面での変更箇所の色付け表示
- JavaScriptに埋め込む動的な値へのエスケープ処理

**未検討**:
- システム利用ユーザ間で共有したい情報を表示できる（例: 「今日は早帰り日です」のようなもの）
- 実行環境がわかる状態で表示できる（例: 単体テスト環境かST環境かを画面表示）
- 表示項目の内容に応じて表示形式の制御ができる（例: 金額項目の右寄せ折り返し表示）
- ヘッダーやフッターなど全画面共通部分のレイアウト定義
- 精査エラー発生時に先頭のエラー項目にフォーカスできる
- エンターキーで入力項目間を移動できる
- エンターキーによるサブミットの許可・不許可を制御できる
- キーバインドをカスタマイズできる（例: F1（ファンクションキー）でサブミット）
- クライアントサイドのバリデーションができる
- ラジオボタンをクリア（未選択な状態）できる
- 画面全体の背景色の指定ができる
- 入力画面の初期フォーカスのデフォルト指定（例: 一番上の入力ボックス）
- 項目名や見出し、入力ボックスなどの背景色・文字（サイズ、色、パディング）の指定ができる
- ボタンのサイズ、色などの指定ができる（標準ボタンの提供）

**取り下げ**:
- 複数エラー発生時のレイアウト崩れ防止表示（フレームワークとしてレイアウト崩れを防ぐ方法を提供できないため取り下げ）
