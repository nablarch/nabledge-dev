# バリデーションとFormの生成

## 概要

バリデーションとForm生成機能の概要。

- **Form**: 外部入力値の入力チェック（1）と外部入力値の保持（2）を担う業務コンポーネント
- 入力値はMap形式、生成するFormはセッタ・ゲッタを持つクラスであること
- バリデーション情報はFormにアノテーションとして設定する
- 本機能がサポートするバリデーション: 単項目の入力チェック（1）、複数項目にまたがる入力チェック（2）
- DBデータを使用した入力チェック（3）はアプリケーションプログラマが別途実装する
- ActionでビジネスロジックのCallの前に実施する入力チェックに使用する（[validation_and_convert](libraries-08_02_validation_usage.md) 以降を参照）

**Entity**: FormをRDBMSテーブルと1対1に対応させ、プロパティとカラムを1対1に対応づけたForm。バリデーションとDB反映の実装が容易になり、RDBMS定義から実装の多くを自動生成できる。

Form以外の業務コンポーネント: [fw-bussiness-component-label](../../about/about-nablarch/about-nablarch-determining_stereotypes.md) 参照。

## 特徴

## アノテーションによる条件の記述

Formに設定したアノテーションをもとにバリデーションを実施する。煩雑な単項目チェック処理の実装が不要となり、バリデーション実装が極小化できる。

バリデーション条件をFormに保持するため、画面処理・バッチ処理など異なる入力元に対してバリデーション条件を共有できる。

## メッセージ機能を使用

バリデーション結果メッセージの作成に [./07_Message](libraries-07_Message.md) の機能を使用。バリデーション結果を表すメッセージとビジネスロジックが発生させるメッセージを透過的に扱える。エラーメッセージ表示処理では、バリデーションで発生したメッセージとその他のビジネスロジックで発生したメッセージを同じように取り扱うことができる。

## 要求: 実装済み

現バージョンで実装済みのバリデーション機能一覧。

- 文字列が特定の文字種のみで構成されているかチェック
- 文字列長チェック
- 必須入力値が入力されているかチェック
- 数値が規定の範囲に入っているかチェック
- 入力値文字列を格納したMapからFormインスタンスを生成
- 入力値文字列がFormプロパティの型に変換可能かチェック
- 入力値文字列をFormプロパティの型に合わせて変換
- Formプロパティ設定前の値に対するバリデーション実行
- エラーメッセージに出力するプロパティ表示名称の任意指定
- エラーメッセージのプロパティ表示名称の国際化
- Form内部プロパティのFormに対するバリデーション
- 日付形式チェック: 西暦(年月日)フォーマット（詳細: :ref:`ExtendedValidation_yyyymmddConvertor`）
- 入力値文字列に対するトリム
- コード値として有効かチェック
