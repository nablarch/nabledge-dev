# Actionクラス内で実行するSQL文の条件網羅テストを簡単に実施する方法はありますか？

## ActionクラスのSQL条件網羅テスト方針

ActionクラスのSQL条件網羅テストは、全テストケースをリクエスト単体テストで実施する必要はない。入力項目数増加によりテストケース数が多くなる場合は、当該ロジックをコンポーネントクラスとして分離し、クラス単体テストで実施すること。

これにより、SQL文やビジネスロジックへの入力変数や参照テーブルを直接セットアップできるため、テスト工数・レビュー工数の削減が期待できる。

クラス単体テストの実施方法は **[プログラミング・単体テストガイド]** -> **[単体テスト実施方法]** -> **[クラス単体テストの実施方法]** -> **[Componentのクラス単体テスト]** を参照。
