# 全体像

## 全体像

Nablarchアプリケーションフレームワークは、実行制御基盤とライブラリから構成される。

**構成要素**:
- **実行制御基盤**: ウェブ、バッチなど処理方式に合わせた実行制御
- **ライブラリ**: データベースアクセス、バリデーションなど個別機能

**特長**:

**様々な処理方式に対応できる**

実行制御基盤と:ref:`ライブラリ<library>`を組み合わせることで、様々な処理方式に対応可能。

**実行制御基盤**:
- :ref:`web_application`
- :ref:`web_service`
- :ref:`batch_application`
- :ref:`messaging`

**すべての実行制御基盤で共通のアーキテクチャを採用**

:ref:`共通アーキテクチャ<nablarch_architecture>`では、パイプライン型の処理モデルに従ってすべてのデータ処理を行う。複数の処理方式を組み合わせて構築するシステムでは、以下のメリットがある。

**柔軟な機能追加・変更**: パイプライン型の処理モデルでは、パイプラインの構成要素であるハンドラの差し替えを容易に行える。機能追加・変更要求に対して柔軟な対応が可能。ハンドラは処理方式間での共有が可能なので、処理方式ごとに同じ機能を重複して作成する必要がない。

**開発方法の共通化**: 各実行制御基盤上で動作するアプリケーションは、ほぼ同様の方法で作成・テスト可能。ある処理方式で開発してスキルを身につけた開発者は、最小限の学習で他の処理方式でも開発できる。これにより、開発者の生産性向上・学習コスト低減を実現でき、開発要員の確保も容易になる。
