# Nablarchのコンセプト

## Robustness

Robustness（堅牢性）: 環境変化・外部攻撃・内部エラーに対しダメージを最小化し機能を維持する能力。Nablarchでは障害耐性・セキュリティ強度・長期使用可能性の3つを含む概念。

## Testability

Testability（テスト容易性）: Nablarchはテスタビリティを重視して設計。各種ドライバ・モック・自動テストフレームワークにより、様々な処理方式・粒度でのテストが可能。

## Ready-to-Use

Ready-to-Use: Nablarchはフレームワークに加え、主要ミドルウェア（Redis、SLF4J等）向けアダプタ、初心者向け開発環境等の再利用可能コンテンツをすぐに使える状態で提供。短期間・高品質なシステム構築を実現。
