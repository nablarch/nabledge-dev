# データバインドと汎用データフォーマットの比較表

この章では、以下の機能の比較を示す。

* データバインド
* 汎用データフォーマット


| 機能 | データバインド | 汎用データフォーマット |
|---|---|---|
| CSVの入出力ができる | ○  解説書へ | ○  解説書へ |
| レコード毎にフォーマットの異なる  CSVの入出力ができる | × [#csv_multi_format]_ | ○  解説書へ |
| CSVの定義を設定できる  (カンマやクォート文字を変更することできる) | ○  解説書へ | ○  解説書へ |
| 固定長データの入出力ができる | ○  解説書へ | ○  解説書へ |
| レコード毎にフォーマットの異なる  固定長データの入出力ができる | ○  解説書へ | ○  解説書へ |
| JSONデータの入出力ができる | × [#json_layout]_ | ○  解説書へ |
| XMLデータの入出力ができる | × [#xml_layout]_ | ○  解説書へ |
| データ入出力時に値の変換ができる  (trimやパック数値やゾーン数値の変換など) | △ [#converter]_ | ○  解説書へ |
| データの寄せ字ができる  システムで許容可能な文字への変換などを指す | × [#char_replace]_ | ○  解説書へ |
レコード毎に異なるフォーマットのCSVを扱う場合には、 汎用データフォーマット を使用すること。
JSONデータの入出力は未実装。JSONデータを扱う場合は、 汎用データフォーマット やOSSを使用すること。
XMLデータの入出力は未実装。XMLデータを扱う場合は、 汎用データフォーマット やJakarta XML Bindingを使用すること。
固定長データのみtrim等のコンバータを提供している。CSVで値を変換したい場合は、出力前及び入力後に変換すること。
入力データの寄せ字(文字変換)は、文字変換用のハンドラを作成し対応すること。
