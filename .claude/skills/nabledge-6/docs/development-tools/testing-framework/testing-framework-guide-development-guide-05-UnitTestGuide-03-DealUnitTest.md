# 取引単体テストの実施方法

## テスト準備

取引単体テストは、アプリケーションサーバにデプロイ後、手動操作でテストを行う方式。

同期応答メッセージ送信処理を伴う場合は、モックアップクラスを使用すること（詳細: :ref:`dealUnitTest_send_sync`）。

テスト準備として以下を行う。

1. データベース準備（データ投入）
2. アプリケーションのデプロイ
3. アプリケーションサーバ起動

## テスト実施

テストケースに従ってテストを実施する。

> **補足**: DBダンプはテスト実行前のものが必要であるため、テスト実施前に取得しておくこと。

## テスト結果エビデンスの収集

収集するエビデンス:
- 画面ハードコピー
- DBダンプ（テスト実行前および実行後）

> **補足**: 画面ハードコピー取得ツール、DBダンプ取得ツール等のテスト補助ツールについては現在検討中。
