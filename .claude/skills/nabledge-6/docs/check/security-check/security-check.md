# セキュリティ実装チェックリスト

## 概要

IPAセキュリティ実装チェックリスト（改訂第7版）およびPCI DSS v3.2.1の要件6に対するNablarchの対応状況を示す。

- **2.チェックリスト**: IPAチェックリストの各項目に対するNablarchの対応状況
- **3.PCIDSS対応表**: PCI DSS要件6.5の各要件とIPAチェックリスト項目のマッピング

> **注意**: 本対応表は単体成果物ではない。プロジェクトのセキュリティガイドラインとともに方式設計のインプットとして使用すること。

> **重要**: PCI DSS要件6以外（要件8等のアプリケーション関連要件）はプロジェクトで個別確認が必要。

**参考**: IPA「安全なウェブサイトの作り方」https://www.ipa.go.jp/security/vuln/websecurity.html

## チェックリスト

IPAセキュリティ実装チェックリストに対するNablarchの対応状況。Nablarchで対応できない項目はプロジェクトで個別対応すること。

**凡例**: ○=対応済、△=一部対応/要プロジェクト実装、×=未対応、※=いずれかを実施

> **重要**: 根本的解決の対策は必ず実施。保険的対策はシステム要件に応じて判断。

| 対策 | 項目 | 実施内容 | Nablarch機能 | 対応 |

| 根本的※ | 1-(i)-b | 文字列連結時、変数をリテラルとして正しく構成 | データベースアクセス（JDBCラッパー）<br>ユニバーサルDAO | ○ |
| 根本的 | 1-(ii) | パラメータにSQL文を直接指定しない | データベースアクセス（JDBCラッパー）<br>ユニバーサルDAO | ○ |

## 6. CSRF（クロスサイト・リクエスト・フォージェリ）（○）

| 対策 | 項目 | 実施内容 | Nablarch機能 | 対応 |
|---|---|---|---|---|
| 根本的※ | 6-(i)-a | 処理実行ページをPOSTでアクセス＋hiddenパラメータに秘密情報を挿入（自動生成）、実行ページで値が正しい場合のみ処理実行 | CSRF対策 | ○ |
| 根本的※ | 6-(i)-b | 処理実行直前のページで再度パスワード入力要求、正しい場合のみ実行 | 6-(i)-aの対策を実施 | - |
| 根本的※ | 6-(i)-c | Refererが正しいリンク元かを確認、正しい場合のみ実行 | 6-(i)-aの対策を実施 | - |
| 保険的 | 6-(ii) | 重要操作時、登録済メールアドレスに自動送信 | - | × |

**説明**: CSRF対策機能は一意トークン発行＋サーバサイドチェックで不正画面遷移を防止（ハンドラ追加のみで漏れなくチェック）。HttpSession使用の二重サブミット防止機能もCSRF対策機能と同等の効果あり（ただし明示的実装が必要で対策漏れの可能性あり）。

> **重要**: データベース使用の二重サブミット防止機能はCSRF対策に非対応。CSRF対策機能を使用すること。

> **注意**: CSRF対策にはCSRF対策機能の使用を推奨（ハンドラ追加のみで漏れなく対策可能）。
