# アプリケーションの責務配置

## アプリケーションの責務配置

Nablarchバッチアプリケーションを作成する際に実装すべきクラスとその責務について説明する。

**アクションクラス (Action Class)**

`DataReader` を生成。読み込んだデータレコードを元に業務ロジック実行し `Result` 返却

例えば、ファイルの取り込みバッチであれば、業務ロジックとして以下の処理を行う:

- データレコードからフォームクラスを作成して、バリデーションを行う
- フォームクラスからエンティティクラスを作成して、データベースにデータを追加する
- 処理結果として `Success` を返す

**フォームクラス (Form Class)**

`DataReader` が読み込んだデータレコードをマッピング。バリデーション用アノテーションと相関バリデーションのロジックを持つ。階層構造(formがformを持つ)も可

**プロパティ定義ルール**: 全て `String` で定義（バイナリ項目はバイト配列）。理由は :ref:`Bean Validation <bean_validation-form_property>` 参照

> **補足**: 外部ファイル等で入力データが安全でない場合のみフォームクラスでバリデーション。データベース等で安全な場合はフォームクラス不要、データレコードから直接エンティティクラス作成

**エンティティクラス (Entity Class)**

テーブルと1対1対応。カラムに対応するプロパティを持つ
