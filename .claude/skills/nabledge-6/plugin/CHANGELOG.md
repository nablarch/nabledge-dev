# 変更履歴

nabledge-6プラグインの主な変更内容を記録しています。

フォーマットは [Keep a Changelog](https://keepachangelog.com/en/1.0.0/) に基づいています。

## [Unreleased]

### 改善
- ワークフローのjq実行を一括処理に最適化し、ツール呼び出し回数を大幅に削減しました
  - キーワード検索: 12回 → 3回（75%削減）
  - セクション判定: 5-10回 → 2-3回（60-70%削減）
  - コード分析: 36回 → 15回（58%削減）
  - 期待されるパフォーマンス（理論値）: キーワード検索 52秒 → 25秒、コード分析 204秒 → 126秒

### 変更
- コード分析の出力先ディレクトリを `.nabledge/` に変更しました。Nabledgeが生成したファイルがプロジェクトのファイルから明確に分離されます（従来は `work/YYYYMMDD/` に出力）

### 改善
- コード分析ワークフローのツール呼び出し数を4回削減し、実行速度を向上させました
  - S3最適化: テンプレートファイル読み込みを3回のReadツール呼び出しから1回のbash catコマンドに統合（2呼び出し削減）
  - S4最適化: 時間計算に関する4回のbash呼び出しを2回に統合（2呼び出し削減）
