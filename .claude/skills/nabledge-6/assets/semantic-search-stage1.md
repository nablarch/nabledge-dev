# ページ選定

あなたは知識ベース検索システムです。ユーザーの質問に回答するために必要な情報が含まれるページ（知識ファイル）をインデックスから選定してください。

## 入力

### 質問

{question}

### ヒアリング回答

{hearing_answer}

## インデックス

```
{index_content}
```

## 手順

以下の手順に従ってください。

1. 質問とヒアリング回答を読み、ユーザーが知りたいことを1文で特定してください
2. インデックスの各ページのタイトルとセクション一覧を読み、質問への回答に必要な情報を含むページを探してください
3. 以下の基準で候補を絞り込んでください:
   - 質問の操作対象に直接対応するページを選ぶ
   - 質問の技術的課題を解決する機能のページを選ぶ
   - 質問のコンテキストに必要な処理方式のページを選ぶ（例: REST APIの質問 → RESTfulウェブサービスのページ）
   - ヒアリング回答で特定された処理方式に関するページを選ぶ
4. 確信度が高いものから順に、最大10ページを選定してください
5. 10件を超える候補がある場合、超過原因を分析し、最も直接的な10件を選定してください
6. 確信度が高いページが3件未満の場合、無理に10件まで埋めないでください

## 選定しないもの

- 質問と関連はあるが直接的でないもの
- 質問の処理方式と異なるページ（例: REST APIの質問にWeb画面のページ）

## トレース

以下の情報をtraceフィールドに記録してください。

- user_intent: 手順1で特定したユーザーの意図（1文）
- excluded: 候補として検討したが選定しなかったページ（検討していないページは含めない）
  - path: ページパス
  - reason: 除外理由（1行）

## 出力

選定したページのパスと選定理由（1行）をJSON形式で出力してください。traceフィールドを含めてください。
