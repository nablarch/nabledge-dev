# ブランクプロジェクト・ドキュメントの刷新について

ブランクプロジェクト・関連ドキュメントの刷新について
ブランクプロジェクトと関連ドキュメントを刷新しました。主な変更点は以下の通りです。

|  | No | 内容 | 詳細 |
|---|---|---|---|
|  | 1 | ブランクプロジェクト構成の変更 | 構成が複雑でセットアップ、管理が困難という意見が多かったため、parent,core,entity,web,batchのプロジェクト構成を見直し、web,batchの構成に変更しました。 |
|  | 2 | ブランクプロジェクトの前提DBの変更 | 初期構築の簡素化のため、初期データベースをOracleデータベースからH2データベース（組込DB）に変更しました。 |
|  | 3 | ブランクプロジェクトの設定ファイルの簡素化 | 環境毎の設定ファイルが初期時点から多く複雑であったため、以下の通り構成変更しました。 ・開発(ut)、結合テスト(it)、総合テスト(st)、本番(production)の4種類用意していましたが、開発(dev)、本番(prod)の2種類に変更しました。 ・固定設定項目(fixed_preset.config,fixed_expansion.config)と共通設定項目(common_preset.config,common_expansion)を統合し、「common.config」としました。 ・環境設定ファイルを統合し、配置場所を変更しました。例えばWeb開発環境の場合、「src/ut/resources/config/system/web_preset.config,web_expansion.config」→ 「src/env/dev/resources/env.config」に変更しました。 また、xmlファイルについてもresourcesフォルダ直下に配置するように変更しました。 |
|  | 4 | ドキュメントの再構成 | 開発リポジトリ構築ガイドの内容を見直し、ブランクプロジェクト、設定変更ガイド、開発リポジトリ構築ガイドに再構成しました。 |
|  | 5 | デフォルト設定一覧の追加 | デフォルトコンフィグレーションで設定している設定値とブランクプロジェクトが設定している設定値の一覧を追加しました。 |
|  | 6 | 使用するデータベースの変更手順の追加 | アーキタイプから生成したブランクプロジェクトの使用するデータベースを変更する手順を追加しました。 |
|  | 7 | 共通プロジェクトの追加手順の追加 | データベースアクセスを伴う共通プロジェクトのサンプル及び追加手順を追加しました。 |
|  | 8 | メール送信バッチの疎通確認手順の追加 | メール送信バッチの疎通確認手順を追加しました。 |
|  | 9 | MOMメッセージング実行基盤の設定手順の追加 | MOMメッセージング実行基盤のサンプル及び設定手順を追加しました。 |
|  | 10 | GSPプラグインの初期設定手順の追加 | GSPプラグインの初期設定手順を追加しました。 |
|  | 11 | マスタデータ復旧機能の使用手順の追加 | マスタデータ復旧機能の使用手順を追加しました。 |
|  | 12 | ツールの組み込み | ブランクプロジェクトに使用許可API一覧作成ツールとJSP静的解析ツールのMavenの設定を組み込み、ブランクプロジェクト構築後すぐに利用可能にしました。 |
|  | 13 | Java7/8でmavenコマンドからFindbugsを動作させる設定のデフォルト組み込みと手順化 | Java7/8を使用した場合、アーキタイプから生成するブランクプロジェクトはmavenコマンド実行時に設定変更しないと、Findbugsが失敗していましたが、デフォルトはJava7/8の利用を想定して設定変更なく動作するように変更しました。また、Java6を使用した場合の設定については手順化しました。 |
|  | 14 | Excelファイルからconfigファイルを自動生成しない変更 | アーキタイプから生成したブランクプロジェクトから環境設定ファイル自動生成ツールの組み込みを外しました。 |
|  | 15 | hiddenタグの暗号化の初期設定を無効に変更 | hiddenタグの暗号化はNablarch5では非推奨であるため、初期設定値を無効に変更しました。 |
