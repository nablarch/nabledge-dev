## UIプラグイン概要とカテゴリ一覧

UI開発基盤の全成果物は**UIプラグイン**単位で管理される。各プラグインは単一の機能・目的に対して作成され、その機能を実現するために必要な成果物一式（タグファイル、JavaScript、CSS、テスト/サンプルコード等）を含む。カスタマイズ時は対応するプラグインをコピーして改修する。

| カテゴリ | 内容 | 主要プラグイン |
|---|---|---|
| 外部ライブラリ | サードパーティ製ライブラリ（カスタマイズ不可） | jQuery(DOM API抽象化)、sugarjs(ネイティブAPI拡張・ES5ポリフィル)、requirejs(動的スクリプトロード・ミニファイ)、fontawesome(アイコン用WebFont) |
| コアJS部品 | JavaScriptで実装された共通処理とスクリプトロード定義（カスタマイズ不可） | スクリプトロード/ドキュメントロード共通処理、Nablarch Submit機能連動カスタムイベント、JavaScriptウィジェット共通基底クラス |

## UIプラグインの構造

各プラグインには機能を実現するためのリソース、ドキュメント、テストが含まれる。以下はカレンダー日付入力ウィジェットプラグインのファイル構成例：

```
nablarch-widget-field-calendar    # プラグイントップ
  ├── package.json                # プラグイン依存関係定義
  ├── bin                         # 開発用コマンド
  ├── ui_local                    # ローカル表示/開発用コマンドソースコード
  │    └── js
  ├── ui_public                   # サーバ動作成果物(サーブレットコンテキスト配下)
  │    ├── css/ui/field-calendar.less
  │    ├── img/
  │    ├── js/nablarch/ui/DatePicker.js, DatePicker.template
  │    └── WEB-INF/
  │          ├── include/
  │          └── tags/widget/field/calendar.tag
  └── ui_test                     # テスト/動作デモ用ソースコード
```

> **重要**: UIプラグインが提供するJSPは原則`WEB-INF`配下に配置する。ただし`ui_test`配下のテスト用JSPは、サーバ稼働確認時にブラウザから直接アクセスするため`WEB-INF`配下ではなく`ui_test`配下のディレクトリに配置する必要がある。

## UIプラグインのバージョンについて

バージョン形式: `(メジャーバージョン).(マイナーバージョン).(パッチ番号)`

| 区分 | 意味 | 初期値 |
|---|---|---|
| メジャーバージョン | 後方互換性を持たない修正に対してインクリメント | 1 |
| マイナーバージョン | 後方互換性を保つ修正（基本的には機能追加）に対してインクリメント | 0 |
| パッチ番号 | 後方互換性を保つ不具合修正に対してインクリメント | 0 |

> **重要**: 後方互換性をもたない修正は通常、個別の派生プラグインとして開発するため、メジャーバージョンは実際の運用では**1**固定となる。
