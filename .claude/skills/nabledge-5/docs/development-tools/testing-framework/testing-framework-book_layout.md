## 本書の構成

本書は5つの章で構成される。最初から通読することも、必要な章のみ個別に読み進めることも可能。

| 章 | タイトル | 内容 |
|---|---|---|
| I | はじめに | 本機能の特徴、採用PJへのメリット・デメリット、採用の前提条件 |
| II | 開発作業手順 | プロジェクトアーキテクトがUI開発基盤に関連して行うタスクの具体的手順 |
| III | プロジェクトのファイル構成と変更管理 | 標準的なファイル構成および変更管理、UI開発基盤側のアップデートの取り込み方法 |
| IV | アーキテクチャ詳説 | UI開発基盤を構成する各機能の技術的な詳細 |
| V | リファレンス | 使用可能なプラグイン、ウィジェット、APIなどの一覧 |
| VI | 補足資料 | その他、開発時に参照すべき資料 |
