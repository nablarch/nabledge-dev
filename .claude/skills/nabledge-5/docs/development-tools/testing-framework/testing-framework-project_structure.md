# 業務画面JSP作成時のディレクトリ構造

**公式ドキュメント**: [業務画面JSP作成時のディレクトリ構造](https://nablarch.github.io/docs/LATEST/doc/development_tools/ui_dev/guide/widget_usage/project_structure.html)

## 業務画面JSP作成時のディレクトリ構造

業務画面JSPのローカル確認用ディレクトリ構造。「ローカル画面確認.bat」を実行して動作を確認する。

| ファイル・ディレクトリ | 内容 |
|---|---|
| WEB-INF | サーバ上の構成と同じで、中に含まれるファイルも原則同じものを使用する。業務画面テンプレート・ウィジェット・業務画面JSPを含む。**業務設計者・業務開発者は直接修正しない** |
| 業務画面JSP | サブシステムIDディレクトリ直下（`ss11AC/`など）に配置。ファイル名は`画面ID.jsp`。**モック動作時はサブディレクトリへの配置不可**。WEB-INF配下に配置 |
| include | 共通領域JSPファイル群。テンプレートからインクルードされる。**業務設計者・業務開発者は直接修正しない**。WEB-INF配下に配置 |
| js | ブラウザ上で動作するスクリプト。**業務設計者・業務開発者は直接修正しない** |
| css | CSSファイル。**業務設計者・業務開発者は直接修正しない** |
| img | 画面表示用画像ファイル。**業務設計者・業務開発者は直接修正しない** |
| fonts | アイコン用フォント。**業務設計者・業務開発者は直接修正しない** |
| tools | UI部品の動作デモ用ツール。**業務設計者・業務開発者は直接修正しない** |
