### I-01: RST ディレクティブ全量調査

**対象仕様**: RST変換ルール（ディレクティブ変換）  
**不明な点**: READMEのディレクティブ変換テーブルは代表例のみ。413ファイルに未記載のディレクティブが使われているか不明。未知ディレクティブが1つでもあればエラー停止する設計のため、事前に全量を把握しないと実装完了できない。

**調査手順**:
1. 全RSTファイルからディレクティブ名（`.. directive_name::` パターン）を正規表現で抽出するスクリプトを作成
2. ディレクティブ名と出現回数を集計
3. READMEの変換テーブルに載っていないものをリストアップ

**スクリプト方針**: `.lw/nab-official/v6/nablarch-document/ja/` 配下の全 `.rst` ファイルをスキャン

**完了基準**:
- 全ディレクティブの一覧と出現頻度が出力できた
- READMEカバー済み / 未カバーの分類が完了した
- 未カバーのものについて対応方針（変換ルール追加 or 除外 or no-content扱い）が決まった

**結果出力先**: 本ファイルの「I-01 結果」セクション

### I-02: RST 見出し記号パターン調査

**対象仕様**: 見出しレベル決定（出現順）  
**不明な点**: READMEは「見出しレベルは出現順で決定」と書くが、RSTの見出し記号（`=`, `-`, `~`, `^`, `"`, `+` 等）の実際の使われ方が不明。overline（上下両方に記号）があるか、3レベル以上（h4以上）が出現するか、ファイルをまたいだ一貫性はあるか。

**調査手順**:
1. 全RSTファイルの見出し行（アンダーライン/オーバーライン）を抽出するスクリプトを作成
2. 使用されている記号パターン（`===`, `---`, `~~~` 等）と、それぞれが何番目に出現するかを集計
3. overline使用ファイルをリストアップ
4. h4以上（4レベル以上の見出し）が出現するファイルをリストアップ

**完了基準**:
- 使用されている見出し記号の全パターンと出現頻度が確認できた
- overline使用の有無と件数が確認できた
- h4以上の見出しが出現するファイルの件数と、それらへの対応方針が決まった

**結果出力先**: 本ファイルの「I-02 結果」セクション

### I-03: クロスリファレンス解決スコープ調査

**対象仕様**: `:ref:`, `:doc:`, `:download:` の解決  
**不明な点**: `:ref:` ラベルはファイルをまたいで参照される。v6全体でのラベル定義数・参照数が不明。外部リポジトリ（Sphinxサブモジュール等）で定義されたラベルへの参照が存在する場合、RBKCでは解決不可能。

**調査手順**:
1. 全RSTファイルから `.. _label:` 定義と `:ref:`ラベル`` 参照を抽出するスクリプトを作成
2. ラベル定義の総数・参照の総数を集計
3. 「定義なし参照」（スキャン対象外ファイルで定義されたラベルへの参照）をリストアップ
4. `:doc:` と `:download:` の参照先が全てスキャン対象内か確認

**完了基準**:
- ラベル定義数・参照数が確認できた
- 解決できない参照（定義なし）の件数と割合が確認できた
- 未解決への対応方針（リンクをテキストに変換 or エラー停止 or スキップ）が決まった

**結果出力先**: 本ファイルの「I-03 結果」セクション

### I-04: RST テーブル複雑ケース調査

**対象仕様**: `list-table` および grid/simple テーブルの変換  
**不明な点**: Markdown テーブルは「空セル」「セル結合（colspan/rowspan）」「複数行セル」に対応していない。RSTの list-table や grid table ではこれらが使用可能。実際のv6ファイルでこれらの複雑ケースが使われているか不明。

**調査手順**:
1. 全RSTファイルから `.. list-table::` の使用箇所を抽出するスクリプトを作成
2. `:header-rows:` `:widths:` 以外のオプション（`:merge:` 等）の使用有無を確認
3. grid table（`+----+----+` 形式）の使用ファイルをリストアップ
4. simple table（`==== ====` 形式）の使用ファイルをリストアップ
5. 空セル・複数行セル・結合セルの実例を抽出

**完了基準**:
- list-table の複雑ケース（空セル・複数行・ネスト）の有無と件数が確認できた
- grid/simple table の使用件数が確認できた
- Markdown変換が困難なケースの対応方針（フォールバック処理等）が決まった

**結果出力先**: 本ファイルの「I-04 結果」セクション
