# Improve search workflow performance

Fixes #98

## Approach

This PR refactors the knowledge search architecture for nabledge-6 and adds nabledge-5 skill based on the work instruction document (`doc/98-improve-search-performance/nabledge-skill-work-instruction.md`).

### Key Changes

**Architecture Redesign:**
- Replace keyword-search → knowledge-search → section-judgement pipeline with unified `_knowledge-search.md`
- Implement fallback strategy: full-text search (route 1) → index-based search (route 2)
- Consolidate section judgement as common workflow for both routes

**Data Format Changes:**
- Section content format: Array of paragraph objects → Single Markdown string
- Index format: Extended to 5-field structure (title, type, category, processing_patterns, path)
- Section-level hints moved from index to knowledge files (`index[].hints`)

**nabledge-5 Addition:**
- Create nabledge-5 skill with same workflow structure as nabledge-6
- Initially 0 knowledge files (will be generated by nabledge-creator)

**Implementation Strategy for This PR:**
- Delete old workflows and scripts only (keep existing 17 knowledge files for testing)
- Implement new workflows and scripts
- Test with existing knowledge files
- After new knowledge files are merged to main, rebase and test with new format
