## 🔬 仮説と改善提案

### 仮説1: v1.2 CA-001のMailRequester欠落は知識ファイルの対象外コード参照が原因

**証拠**: W11AC02Action.java内のメール送信コードがanalyzedされず、Overview/Processing Flow/Sequence diagramに全く現れない。エージェントが選択した知識ファイルはweb-application-07_insertとweb-application-04_validationのみで、メール送信関連知識（SyncMessage等）を読み込んでいない。

**検証方法**: エージェントが読み込んだ知識ファイルのログを確認し、messaging/SyncMessage関連知識ファイルが存在するか確認する。

**期待結果**: MailRequester/MessageSender対応の知識ファイル追加で検出率向上（ca-001: 72.2% → 80%超が見込まれる）。

### 仮説2: Overview検出ミスはCA実装が高レベル要約を優先するため

**証拠**: BusinessDateUtil、UserInfoTempEntityはclassDiagramとComponentSummaryには正しく登場するが、Overviewの2段落サマリーには登場しない。v1.4/v1.3でも同様のパターン（OverviewでValidatableFileDataReader等が省略）が観察されている。

**検証方法**: Overviewのプロンプト指示を確認し、"重要クラスを明示的に列挙する"よう指示できるか確認する。

**期待結果**: Overview detection率を改善するためにはCA workflowのOverview生成プロンプトを修正する必要がある（全バージョン共通の構造的問題）。
