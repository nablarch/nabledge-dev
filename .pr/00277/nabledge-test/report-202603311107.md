## 🔬 仮説と改善提案

### 1. Overview class list差異の原因: 知識検索ヒットの違い

**仮説**: v1.2の知識ファイルが`MailRequester`/`ValidatableFileDataReader`等をv1.3より少ない頻度で参照しているため、知識検索でこれらが上位にヒットせず、Overview生成時に抽出されない。

**検証方法**: v1.2とv1.3の full-text-search.sh に同じクエリ（`MailRequester`, `MessageSender`, `ValidatableFileDataReader`）を実行して、ヒット数・スコアを比較する。

**期待効果**: もし差異があれば、v1.2知識ファイルの当該クラスの説明を強化することで Overview に含まれるようになる可能性がある。

### 2. DbAccessSupport Nablarch Usage の安定化

**仮説**: DbAccessSupportはスーパークラスであるが、エージェントがNablarch Usageセクションに含めるかを毎回判断している（ランダム性あり）。今回はValidationUtil/OnDoubleSubmission/MailRequester/MessageSenderを選択した。

**検証方法**: 複数試行を実施して検出率を測定（ベースラインモードで3試行）。

**改善方向**: flow_content/nablarch_usage の guideline に「継承元Nablarchクラスも含める」を明示することで安定化が見込める。
