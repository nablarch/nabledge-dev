## 💡 主要な発見

### 1. qa-001 の系統的な低スコア（50〜62.5%）

qa-001「コード値のプルダウン入力」はベンチマークシナリオとして3試行を実施したが、全試行で50〜62.5%と低水準で安定している。検出できないのは `n:select`（汎用セレクトタグ）、`listName`、`elementValueProperty`/`elementLabelProperty`（OR条件）、`withNoneOption` の4項目。

v5スキルは `n:codeSelect` を中心とした回答を生成しており、コード値専用タグの説明は充実しているが、**汎用の `n:select` タグと空オプション追加（withNoneOption）をカバーしていない**。これは nabledge-5 の知識ファイルに `n:select` の `listName`/`elementValueProperty` 関連のコンテンツが少ない可能性を示唆する。

### 2. qa-002 の `pageNumber`/`listSearchResult` 未検出

qa-002「ページング検索」は6/8（75%）。`Pagination`/`getPagination` は検出できているが、`pageNumber`（ページ番号リンクのJSPタグ）と `listSearchResult`（表示件数/ページリンク）が未検出。これはJSP側のタグライブラリ情報をスキルが出力していないことを示す。バックエンドAPIの説明に偏り、フロントエンドのUI実装部分が欠ける傾向がある。

### 3. ca-002 の Nablarch Framework Usage セクション構造の不一致

ca-002「ImportZipCodeFileAction」のNablarch Framework Usage ヘッダ検出（`### BatchAction`, `### UniversalDao`, `### BeanUtil`, `### DataReader`）が全試行で0/4（0%）。スキルはNablarch利用をまとめて記述するが、各クラスを `###` 見出しで独立したセクションとして記述しない。また、ca-002 trial 2 では prefill-template.sh の `{{output_path}}` sed バグにより完全なアウトプット生成に失敗（3.1%）。これはスクリプトのバグが再現性を持って影響していることを示す。

## 🔬 仮説と改善提案

### 仮説1: `n:select` + `listName` 知識ファイルの欠如
**証拠**: qa-001 で3試行とも `n:select`/`listName` 未検出。
**検証方法**: `libraries-tag.json` に `n:select` の `listName`/`elementValueProperty` 説明が含まれているか確認。
**期待される結果**: 知識ファイルに内容を追加すれば qa-001 を 75〜87.5% に改善できる可能性がある。

### 仮説2: Nablarch Framework Usage の見出し形式が仕様と不一致
**証拠**: ca-002 の全3試行で `BatchAction`/`UniversalDao`/`BeanUtil`/`DataReader` が `###` 見出しとして出力されない。ca-001 では `### UniversalDao`/`### SessionUtil` 等が正常に出力されている。
**検証方法**: ca-002 の出力ファイルで実際の見出し構造を確認する。「メインアクションが依存するNablarchクラス群」だけでなく「バッチ基盤クラス（BatchAction等）」が独立セクションとして記述されているか確認。
**期待される結果**: スキルのプロンプトに「BatchActionを含む継承元クラスも独立した `###` セクションで記述する」旨を追加することで改善できる可能性がある。

### 仮説3: prefill-template.sh の `{{output_path}}` バグが試行安定性を損なう
**証拠**: ca-002 trial 2 が完全失敗（3.1%）。バグは既知でワークアラウンドが存在するが、エージェントが毎回ワークアラウンドを必要とするため試行間の一貫性が低下する。
**検証方法**: `escape_sed()` 関数の `/` のエスケープ処理を修正する（`s/[&/\\[\\]*.\\^$]/\\\\&/g` → フォワードスラッシュを `|` デリミタ回避で対処）。
**期待される結果**: スクリプト修正後は ca-002 の成功率が安定して 78%+ を維持できる。
