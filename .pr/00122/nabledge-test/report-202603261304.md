## 💡 主要な発見

### 1. 検出ギャップはランダムではなく構造的

全シナリオで未検出となった項目はいずれも**期待値の設計上の問題**に起因する。QAでは:
- **qa-001**: `n:select` が未検出 — AIは一貫して `n:codeSelect` タグのみを回答に使用する（`n:select` はv1.4 RST に存在しない可能性が高い）
- **qa-003**: `-requestPath` が未検出 — AIは `requestPath` をオプション名として正しく説明するが、CLIの `-` プレフィックス付き形式では記述しない

CAでは:
- **`.nabledge/` パス**: 全CAシナリオで未検出 — 出力ファイルのコンテンツ内にパスが記述されないため、ファイル存在確認ではなくテキスト検索では検出できない構造的問題
- **Processing Flow / Sequence の private メソッド**: `sendMailToRegisteredUser` / `checkLoginId`（ca-001）、`getValidatorAction`（ca-002）— フローの叙述文では呼び出し元視点のpublicメソッドのみ記述され、private実装詳細は省略される

### 2. v1.4 初回ベースライン: 85.6% (QA 90.0% / CA 84.3%)

スコア分布は均等で、low detectionのアウトライヤーはない:
- 最低: qa-001 / qa-003 各 75% — 両方とも1項目欠落のみ
- CA: ca-001 86.1% / ca-002 82.4% — Overviewセクションが詳細クラス名より高レベルな要約になることによる欠落（ValidatableFileDataReader/BusinessDateUtil/UserInfoTempEntity が Overview に出ない）

v6ベースライン(94.6%)との差は主に期待値の精度に起因すると考えられる。

### 3. CAパフォーマンス: WebアクションはBatchより60%遅い

- ca-001 (W11AC02Action, Web): 320秒 — 6メソッド×複数送信手段（DB/MOM/HTTP）の複雑度
- ca-002 (B11AC014Action, Batch): 202秒 — FileBatchAction継承の直線的パターン

両方とも**ドキュメント生成ステップが全体時間の65%以上**を占める（QAの「候補セクション読込み」が27%と対照的）。CAはより長い出力を生成するため、生成時間が支配的になる。

## 🔬 仮説と改善提案

### 仮説1: `n:select` 期待値の修正で qa-001 を 100% にできる

**根拠**: v1.4 の RST ドキュメントには `n:select` が存在しない（v5/v6 で同様に修正済み）。AIは `n:codeSelect` タグのみを正しく回答している。

**検証方法**: `.claude/skills/nabledge-1.4/` のナレッジファイルを grep して `n:select` の出現頻度を確認。

**期待される結果**: `n:select` を期待値から削除することで qa-001 の検出率が 75% → 100% に改善。

### 仮説2: `.nabledge/` のOutput検出は別の確認手段が必要

**根拠**: 3試行のCAシナリオすべてで `.nabledge/` が Output に未検出。スキルがファイルを `.nabledge/` ディレクトリに正しく書き出していても、テキスト内にパスが記述されない限り検出できない。

**検証方法**: nabledge-test の grading スクリプトで Output 検出を「ファイルが .nabledge/ ディレクトリに存在するか」で判定するよう修正。

**期待される結果**: `.nabledge/` の実際の書き出し状況を正確に測定できるようになり、CAの検出率が +2.9% 改善（34/34 = 100% も可能）。

### 仮説3: Overview に詳細クラス名を含めることで CA スコアを改善できる

**根拠**: ca-002 の Overview 未検出3項目（ValidatableFileDataReader, BusinessDateUtil, UserInfoTempEntity）はいずれもコード分析ワークフローの Overview セクションが「高レベル要約」を指示しているため自然に省略される。

**検証方法**: code-analysis ワークフローの Overview 生成指示を「主要 Nablarch クラスを1〜2個 Overview に明示する」よう追加修正してから再テスト。

**期待される結果**: ca-002 の Overview スコアが 2/5 → 4-5/5 に改善し、全体検出率が +5% 程度向上。
